# Открытые линии: типовые сценарии

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

Сценарий — это последовательность запросов для одной задачи в открытых линиях. Сценарии помогают подключать внешние каналы, передавать обращения клиентов в Битрикс24 и возвращать ответы операторов в тот же канал.

Открытая линия задает правила обработки обращения внутри Битрикс24: очередь операторов, маршрутизацию, режим работы и другие настройки диалога. Коннектор связывает открытую линию с внешним каналом, например чатом на сайте или другим сервисом.

## Связь с другими объектами

Сценарий с внешним чатом строится вокруг коннектора, внешнего чата и обработчика приложения.

- **Коннектор.** Коннектор связывает внешний канал с открытой линией. Приложение регистрирует его методом [imconnector.register](../../api-reference/imopenlines/imconnector/imconnector-register.md), а затем Битрикс24 показывает коннектор в настройках контакт-центра. При регистрации задайте код коннектора в параметре `ID`, затем передавайте это значение в параметре `CONNECTOR`
- **Внешний чат.** Внешний сайт или сервис хранит публичную часть диалога: интерфейс чата, идентификатор посетителя и историю сообщений. Сообщение клиента передается в Битрикс24 методом [imconnector.send.messages](../../api-reference/imopenlines/imconnector/imconnector-send-messages.md). В массиве `MESSAGES` приложение передает данные клиента, сообщения и чата
- **Обработчик приложения.** Обработчик принимает события от Битрикс24. Когда оператор отвечает из открытой линии, приложение получает событие [OnImConnectorMessageAdd](../../api-reference/imopenlines/imconnector/events/on-im-connector-message-add.md), отправляет ответ во внешний чат и подтверждает доставку методом [imconnector.send.status.delivery](../../api-reference/imopenlines/imconnector/imconnector-send-status-delivery.md). В [imconnector.register](../../api-reference/imopenlines/imconnector/imconnector-register.md) передайте `PLACEMENT_HANDLER` для интерфейса настройки коннектора, а в [event.bind](../../api-reference/events/event-bind.md) передайте `handler` для события

## Как начать работу

1. Определите внешний канал: чат на сайте, форму связи или другой сервис, где клиент начинает диалог
2. Создайте приложение Битрикс24 и подготовьте публичные URL с доступом по HTTPS: для интерфейса настройки коннектора и для обработчика событий
3. Зарегистрируйте коннектор методом [imconnector.register](../../api-reference/imopenlines/imconnector/imconnector-register.md)
4. Подпишите приложение на событие [OnImConnectorMessageAdd](../../api-reference/imopenlines/imconnector/events/on-im-connector-message-add.md) методом [event.bind](../../api-reference/events/event-bind.md)
5. Подключите коннектор к нужной открытой линии в Битрикс24. В обработчике настройки приложения вызовите [imconnector.activate](../../api-reference/imopenlines/imconnector/imconnector-activate.md) и передайте `ACTIVE=1`
6. Передавайте сообщения клиента методом [imconnector.send.messages](../../api-reference/imopenlines/imconnector/imconnector-send-messages.md)
7. Принимайте ответы оператора в обработчике события и подтверждайте доставку методом [imconnector.send.status.delivery](../../api-reference/imopenlines/imconnector/imconnector-send-status-delivery.md)

## Что важно учитывать

- Коннектор работает через приложение. Вебхуки не подходят: приложению нужно зарегистрировать коннектор, принимать события и работать с настройками открытой линии
- Для методов коннектора и подписки на событие [OnImConnectorMessageAdd](../../api-reference/imopenlines/imconnector/events/on-im-connector-message-add.md) приложению нужен scope [`imopenlines`](../../api-reference/scopes/permissions.md)
- Используйте один и тот же код коннектора при регистрации, активации, отправке сообщений и обработке событий
