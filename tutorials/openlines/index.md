## Коннектор открытых линий

Коннектор связывает внешний канал с открытой линией. Внешний сервис передает сообщения клиентов в Битрикс24. Ответы сотрудников возвращаются обратно в этот сервис.

Подход подходит для случаев, когда интеграция управляет публичной частью чата и хранит состояние диалога на своей стороне. Сайт отправляет сообщения через метод [imconnector.send.messages](../../api-reference/imopenlines/imconnector/imconnector-send-messages.md). Обработчик событий принимает ответы из Битрикс24 и подтверждает доставку через [imconnector.send.status.delivery](../../api-reference/imopenlines/imconnector/imconnector-send-status-delivery.md).

## Этапы подключения

Статья про коннектор описывает полный цикл подключения внешнего чата.

1. Регистрирует коннектор в Битрикс24 методом [imconnector.register](../../api-reference/imopenlines/imconnector/imconnector-register.md).
2. Подписывает приложение на событие [OnImConnectorMessageAdd](../../api-reference/imopenlines/imconnector/events/on-im-connector-message-add.md) о новых сообщениях.
3. Активирует коннектор для выбранной открытой линии.
4. Передает сообщения посетителя с сайта в Битрикс24.
5. Принимает ответы оператора и подтверждает доставку.

## Технические требования

- Используйте авторизацию через приложение. В примере используется [CRest PHP SDK](../../sdk/crest-php-sdk/index.md). Вебхуки не подходят, потому что коннектор регистрируется как часть приложения, принимает события и работает с настройками открытой линии.
- Разместите обработчик событий на сервере с доступом по протоколу HTTPS.
- Храните историю диалогов в базе данных для обеспечения надежности.
- Зафиксируйте идентификатор коннектора и используйте его при регистрации, отправке сообщений и обработке событий.
