# Получение списков в CRM

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Списки CRM — это выборки данных, которые нужны в интеграциях: дубликаты по телефону и email, дела по сделкам, стадии, воронки, адреса, поставщики и элементы на выбранной стадии.

Сценарий — это последовательность запросов для одной задачи. В нем описан порядок вызова методов и приведен пример кода.

> Быстрый переход: [все сценарии](#choose-tutorial)

## Связь с объектами CRM

Сценарии связаны с клиентами, сделками, смарт-процессами и складским учетом.

- **Лиды, контакты и компании.** Дубликаты ищут по телефону и email, затем получают данные [лидов](../../../api-reference/crm/leads/index.md), [контактов](../../../api-reference/crm/contacts/index.md) и [компаний](../../../api-reference/crm/companies/index.md). Адрес клиента хранится в [реквизитах](../../../api-reference/crm/requisites/index.md)
- **Сделки.** Список дел строят по `ID` сделок. Стадии и воронки помогают отобрать сделки для следующих действий в CRM. Базовые операции выполняют методами [crm.deal.*](../../../api-reference/crm/deals/index.md)
- **Смарт-процессы.** Элементы фильтруют по стадиям через `entityTypeId` — идентификатор [типа объекта CRM](../../../api-reference/crm/data-types.md#object_type). Получить элементы можно универсальными методами [crm.item.*](../../../api-reference/crm/universal/index.md)
- **Поставщики.** Поставщиков получают из контактов или компаний по категории. Полученные идентификаторы передают в метод складского учета [catalog.documentcontractor.add](../../../api-reference/catalog/documentcontractor/catalog-documentcontractor-add.md)

## Как начать работу

1. Определите объект CRM: лид, контакт, компания, сделка или смарт-процесс
2. Найдите нужный сценарий в таблице [Как выбрать сценарий](#choose-tutorial)
3. Проверьте, какие права и scopes указаны в выбранном сценарии
4. Выполните методы в порядке, который описан в сценарии
5. Используйте полученные идентификаторы в следующих запросах CRM или складского учета
