{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

Сквозная аналитика помогает понять, из какого источника пришел клиент и что он сделал до обращения: перешел по рекламе, открыл страницы сайта, заполнил форму или позвонил. В CRM эти данные можно связать с лидом, сделкой, контактом, компанией или коммерческим предложением.

Например, клиент перешел на сайт из рекламной кампании, посмотрел несколько страниц и отправил форму. Если передать данные сквозной аналитики в CRM, менеджер увидит источник обращения, а в отчетах будет видно, какая реклама привела клиента и с какой сделкой она связана.

Туториалы показывают, как передать данные аналитики при создании объектов CRM или связать уже созданные объекты с одним трейсом. Выберите практический сценарий в таблице ниже.

> Быстрый переход: [все сценарии](#choose-tutorial)

## Связь с объектами CRM

Данные сквозной аналитики можно передать пяти объектам CRM: лиду, контакту, компании, сделке и коммерческому предложению. Есть два способа:

- **При создании объекта** — вместе с его полями в методе создания. Источник обращения передают через UTM-поля, полный маршрут клиента — через поле `TRACE`, если метод создания поддерживает это поле. Для универсального метода [crm.item.add](../../../api-reference/crm/universal/crm-item-add.md) полный трейс связывают отдельным методом [crm.tracking.trace.add](../../../api-reference/crm/tracking/crm-tracking-trace-add.md). Какие поля доступны для объекта, смотрите в [туториале](./info-to-analitics.md)
- **Связыванием уже созданных объектов** — методом [crm.tracking.trace.add](../../../api-reference/crm/tracking/crm-tracking-trace-add.md). В параметре `ENTITIES` передают типы объектов — `LEAD`, `CONTACT`, `COMPANY`, `DEAL`, `QUOTE` — и их идентификаторы. Так с одним трейсом можно связать как один объект, так и несколько

## Как начать работу

1. Выберите сценарий в таблице [Как выбрать сценарий](#choose-tutorial)
2. Подготовьте форму на сайте для сбора данных клиента
3. Если нужен полный маршрут клиента, получите `TRACE` через `b24Tracker.guest.getTrace()`. Передайте его при создании объекта, если метод поддерживает поле `TRACE`, или свяжите объект с трейсом через [crm.tracking.trace.add](../../../api-reference/crm/tracking/crm-tracking-trace-add.md). Для объектов, созданных универсальным методом [crm.item.add](../../../api-reference/crm/universal/crm-item-add.md), используйте [crm.tracking.trace.add](../../../api-reference/crm/tracking/crm-tracking-trace-add.md). Если достаточно источника обращения, передайте UTM-поля
4. Проверьте, какие права и scopes указаны в выбранном сценарии
5. Выполните методы в порядке, который описан в сценарии
