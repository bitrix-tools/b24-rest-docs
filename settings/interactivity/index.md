# Интерактивность в приложениях: обзор сценариев и методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Интерактивность помогает приложению работать без перезагрузки страницы: показывать изменения в интерфейсе, обновлять статус и отправлять push-уведомления пользователям.

Это нужно, чтобы пользователь сразу видел изменения и не обновлял страницу вручную. Например, сотрудник может запустить проверку документа и после завершения увидеть новый статус. Если сотрудник не у компьютера, приложение может отправить push-уведомление о результате на телефон.

{% note warning %}

В коробочной версии интерактивность в приложениях доступна начиная с [версии](../cloud-and-on-premise/on-premise/versions.md) модуля Push&Pull 18.5.500. Если каналы или методы недоступны, сначала проверьте версию модуля.

{% endnote %}

## Как выбрать сценарий

В Битрикс24 для работы с интерактивностью есть два основных сценария: приложение может получать события через встроенный Push&Pull в браузере или через собственный клиент. Выбор зависит от того, где должно работать приложение и как оно будет получать события.

**Push&Pull в браузере.** Подойдет, если приложение открыто в интерфейсе Битрикс24 и должно получать события в браузере. Например, пользователь работает с документом в приложении. Серверная часть завершает проверку и сразу отправляет событие в браузер, чтобы на экране появился новый статус. [Push&Pull в браузере](./push-and-pull-in-browser.md).

**Собственный Push&Pull клиент.** Используют, если приложению недостаточно встроенной работы в браузере. Такой вариант подходит, если приложение работает отдельно от интерфейса Битрикс24 и должно поддерживать соединение с серверами реального времени, получать события и восстанавливать подключение при обрыве. [Собственный Push&Pull клиент](./custom-push-and-pull-client.md).
