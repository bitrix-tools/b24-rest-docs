# Установка локальных приложений: обзор сценариев

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

Локальное приложение работает внутри одного Битрикс24 — там, где его создали. Оно может показывать отчеты, обмениваться данными с внешней системой или добавлять собственные элементы интерфейса.

После сохранения приложение добавляется в Битрикс24, а дальнейшая установка зависит от выбранного сценария. Перед сохранением выберите сценарий:

- сразу сделать приложение доступным пользователям
- сначала показать администратору страницу первоначальной настройки
- передать данные авторизации на сервер приложения

> Быстрый переход: [как выбрать сценарий](#installation-scenarios)
>
> Пользовательская документация: [Разработчикам: как создать вебхуки и приложения для Битрикс24](https://helpdesk.bitrix24.ru/open/20886106/#2)

## Связь с другими объектами

**OAuth 2.0.** Битрикс24 передает приложению данные авторизации, с которыми оно выполняет методы. Получение и обновление токенов описано в разделе [OAuth 2.0](../../oauth/index.md).

**События и виджеты.** Во время первоначальной настройки приложение может зарегистрировать [обработчики событий](../../../api-reference/events/index.md) или добавить [виджеты](../../../api-reference/widgets/index.md) в интерфейс Битрикс24.

## Как начать работу

1. Определите, нужен ли приложению интерфейс в Битрикс24
2. Подготовьте страницу приложения, серверный обработчик или ZIP-архив со статическими файлами
3. Выберите вариант в таблице [Как выбрать сценарий](#installation-scenarios)
4. Откройте форму локального приложения по [пользовательской инструкции](https://helpdesk.bitrix24.ru/open/20886106/#2)
5. Укажите название, выберите права доступа и заполните поля выбранного сценария
6. Сохраните приложение и выполните первоначальную настройку, если она предусмотрена
