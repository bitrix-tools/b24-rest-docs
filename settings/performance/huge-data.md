# Как получить большие объемы данных

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

При импорте большого числа элементов через списочные методы со стандартной REST-навигацией не используйте обычную постраничную навигацию с последовательным увеличением `start = start + 50`.

Вместо этого выбирайте страницы через фильтр по последнему полученному идентификатору и передавайте `start = -1`. Такой подход отключает подсчет общего количества элементов и снижает нагрузку на Битрикс24.

При `start >= 0` стандартная REST-навигация возвращает страницу данных и считает общее количество элементов для поля `total`. На больших выборках или сложных фильтрах подсчет может работать медленно.

Для одной страницы данных фильтр по последнему идентификатору не нужен. Достаточно передать `start = -1`, чтобы метод вернул до 50 элементов без подсчета `total`.

Обычная навигация списочных методов через поля `next` и `start` описана в статье [Особенности работы списочных методов](../how-to-call-rest-api/list-methods-pecularities.md).

## Как выбрать все элементы

Чтобы последовательно получить все элементы по фильтру, используйте методы, которые поддерживают сортировку по стабильному числовому идентификатору.

1. Отсортируйте элементы по идентификатору по возрастанию
2. Добавьте в фильтр условие `>ID` или `>id` со значением последнего полученного идентификатора
3. Передайте `start = -1`, чтобы отключить подсчет общего количества
4. После каждого запроса сохраните идентификатор последнего элемента из ответа
5. Повторяйте запрос, пока метод не вернет меньше 50 элементов

Такой цикл не использует поле `next`. Следующая страница формируется фильтром по идентификатору.

## Что учесть при импорте

Между запросами оставляйте задержку, чтобы не превысить [лимиты REST API](limits.md). Если метод возвращает ошибку лимита, уменьшите частоту запросов и повторите вызов позже.

{% note info "" %}

Применяйте этот подход только для методов, в которых фильтр и сортировка по идентификатору работают со `start = -1` как стандартная REST-навигация. Если во время импорта элементы удаляются или права доступа меняются, состав выборки может измениться между запросами.

{% endnote %}

Ниже приведен пример кода и сравнение времени выполнения с обычной постраничной навигацией. При 2 387 743 элементах с одинаковыми правами и фильтром время выполнения уменьшилось с 49,9 секунды до 0,097 секунды.

## Пример

```php
$tokenID = 'XXXXXXXXXXXXXXXXXXXXX';
$host = 'XXXX.bitrix24.ru';
$user = 1;

// Начинаем с нуля или с ID, на котором остановился предыдущий импорт.
$lastId = 0;
$finish = false;

while (!$finish)
{
    $http = new \Bitrix\Main\Web\HttpClient();
    $http->setTimeout(5);
    $http->setStreamTimeout(50);

    $json = $http->post(
        'https://'.$host.'/rest/'.$user.'/'.$tokenID.'/crm.item.list/',
        [
            'entityTypeId' => 1,
            'order' => ['id' => 'ASC'],
            'filter' => ['>id' => $lastId],
            'select' => ['id', 'title', 'createdTime'],
            'start' => -1,
        ]
    );

    $result = \Bitrix\Main\Web\Json::decode($json);
    $items = $result['result'] ?? [];
    $itemsCount = count($items);

    if ($itemsCount === 0)
    {
        break;
    }

    foreach ($items as $item)
    {
        $lastId = $item['id'];
        // Обработайте элемент.
    }

    if ($itemsCount < 50)
    {
        $finish = true;
    }

    if (!$finish)
    {
        usleep(500000);
    }
}
```
