# BX24.js: обзор библиотеки

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

BX24.js — JavaScript-библиотека для приложений, встроенных в интерфейс Битрикс24. Она позволяет вызывать методы Битрикс24 из клиентской части приложения и автоматически добавляет данные авторизации OAuth 2.0.

Библиотека также связывает интерфейс приложения с интерфейсом Битрикс24. С ее помощью можно управлять размером фрейма, открывать стандартные диалоги и страницы Битрикс24, получать данные о среде выполнения, хранить настройки пользователя и приложения.

> Быстрый переход: [обзор страниц BX24.js](#all-pages)

## Как начать работу

1. Подключите библиотеку на странице приложения:

   ```html
   <script src="//api.bitrix24.tech/api/v1/"></script>
   ```

2. Дождитесь инициализации библиотеки с помощью [BX24.init](./system-functions/bx24-init.md)
3. Выберите группу функций в таблице [Обзор страниц BX24.js](#all-pages)
4. Вызовите нужную функцию и обработайте результат в клиентском коде приложения

## Связь с другими инструментами

**Места встраивания.** Приложение можно открыть в выбранном месте интерфейса Битрикс24. Для регистрации обработчика используйте метод [placement.bind](../../api-reference/widgets/placement-bind.md): передайте код места встраивания в параметре `PLACEMENT` и внешний URL обработчика виджета в параметре `HANDLER`. Метод доступен администратору.

**UI Kit.** BX24.js отвечает за взаимодействие приложения с Битрикс24, а [Битрикс24 UI Kit](../../api-reference/widgets/ui-kit/index.md) помогает собрать интерфейс из готовых компонентов, дизайн-токенов и иконок.

## Что важно учитывать

- Для внешних приложений и вебхуков выберите другую библиотеку в [обзоре SDK](../index.md)
- Необходимый scope зависит от вызываемого метода. Значения scope смотрите в справочнике [Права доступа](../../api-reference/scopes/permissions.md)
- Доступ к данным зависит от прав пользователя, от имени которого приложение выполняет запрос
