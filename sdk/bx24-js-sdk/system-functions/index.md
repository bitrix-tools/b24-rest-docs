# Инициализация и авторизация: обзор функций

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Системные функции подготавливают встроенное приложение к работе. Их используют до вызовов API и работы с интерфейсом.

{% note info "" %}

Функции раздела работают только в контексте [приложения](../../../settings/app-installation/index.md).

{% endnote %}

> Быстрый переход: [все функции](#all-methods)

## Как начать работу

1. Дождитесь готовности библиотеки через [BX24.init](./bx24-init.md), чтобы получить данные Битрикс24 и перейти к следующим вызовам
2. Зарегистрируйте обработчик через [BX24.install](./bx24-install.md), если нужен сценарий первого запуска приложения у текущего пользователя
3. Завершите настройку вызовом [BX24.installFinish](./bx24-install-finish.md), чтобы перейти к обычной инициализации приложения
4. Получите текущие OAuth-данные через [BX24.getAuth](./bx24-get-auth.md) или обновите их через [BX24.refreshAuth](./bx24-refresh-auth.md)

## Обзор функций {#all-methods}

### Инициализация и первый запуск

#|
|| **Функция** | **Описание** ||
|| [BX24.init](./bx24-init.md) | Добавляет обработчик события «библиотека готова к работе» ||
|| [BX24.install](./bx24-install.md) | Регистрирует обработчик первого запуска приложения у текущего пользователя ||
|| [BX24.installFinish](./bx24-install-finish.md) | Завершает работу инсталлятора или настройщика приложения ||
|#

### Авторизация приложения

#|
|| **Функция** | **Описание** ||
|| [BX24.getAuth](./bx24-get-auth.md) | Получает текущие данные для авторизации через OAuth 2.0 ||
|| [BX24.refreshAuth](./bx24-refresh-auth.md) | Обновляет ключ авторизации принудительно ||
|#
