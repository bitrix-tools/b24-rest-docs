# Универсальные списки: обзор методов

Универсальные списки — это инструмент для создания и управления структурированной информацией. Они позволяют создавать настраиваемые таблицы c различными типами полей: текст, число, дата, файл и другие.

Универсальные списки интегрируются с модулями Битрикс24, что дает возможность автоматизировать обработку данных и управлять доступом. Представьте, что есть список заявок на отпуск сотрудников. Для списка настроен бизнес-процесс и заданы права так, что только руководитель и сотрудники HR-отдела могут изменять заявки. В результате, когда сотрудник подает заявку на отпуск, HR-менеджер автоматически получает уведомление, чтобы рассмотреть и одобрить или отклонить заявку.

Каждый список представляет собой [информационный блок](*iblock). Используйте группу методов [lists.*](./lists/index.md) для работы со списками.

## Структура универсального списка

### Разделы

Разделы группируют элементы, как папки. Это помогает выстроить иерархию. Например, раздел «Юридические документы» -> подраздел «Договоры аренды». Для работы с разделами используйте методы [lists.section.*](./sections/index.md).

### Элементы

Элементы — это строки списка, которые хранят основную информацию. Например, данные одного договора, заявки или сертификата. Элемент можно поместить в раздел. Методы для работы с элементами — [lists.element.*](./elements/index.md).

{% note tip "Пользовательская документация" %}

{% endnote %}

### Поля

Поля определяют, какие данные хранятся в каждом элементе. Вы задаете тип поля: текст, список, дата, файл и другие. Создавать, изменять и удалять поля можно через методы [lists.field.*](./fields/index.md).

Набор полей уникален для каждого списка. Все доступные типы смотрите в описании параметра `FIELDS` метода [lists.field.add](./fields/lists-field-add.md).

Информация в универсальных списках может быть связана с CRM и Диском через поля следующих типов:

-  Привязка к элементам CRM,
-  Файл (Диск).

Об особенностях этих типов читайте в статье [Поля универсального списка: обзор методов](./fields/index.md).

## Последовательность работы со списком

Рекомендуемый порядок работы со списком:

1. Создайте список с помощью метода [lists.add](./lists/lists-add.md). Укажите код `IBLOCK_CODE`, тип `IBLOCK_TYPE_ID` и название `NAME` списка.

2. Добавьте пользовательские поля через метод [lists.field.add](./fields/lists-field-add.md), чтобы задать структуру данных будущих элементов.

3. Если нужно, создайте разделы методом [lists.section.add](./sections/lists-section-add.md) для организации иерархии.

4. Наполните список элементами с помощью метода [lists.element.add](./elements/lists-element-add.md). Заполняйте созданные поля и привязывайте элементы к разделам.
