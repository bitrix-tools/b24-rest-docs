# Элементы универсального списка: обзор методов

Каждый элемент списка состоит из полей. Вы можете добавлять новые поля и настраивать их методами [lists.field.\*](../fields/index.md).

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Как добавить элементы списка](https://helpdesk.bitrix24.ru/open/5405819)

## Связь с другими объектами

**Универсальные списки.**  Элементы создаются внутри списков. Чтобы добавить элемент в список, укажите параметры `IBLOCK_TYPE_ID`, `IBLOCK_ID` и `IBLOCK_CODE`. Значения можно получить методом [lists.get](../lists/lists-get.md).

**Разделы  списка.** Элемент можно поместить в раздел с помощью параметра `IBLOCK_SECTION_ID`. Идентификатор раздела можно получить методом [lists.section.get](../sections/lists-section-get.md).

**Элементы других списков.** Поле типа Привязка к элементам связывает элементы разных списков. В таком поле можно выбрать другой список и указать его элементы как значения. Внутри поля сохраняются `ID` выбранных элементов, а отображаются их названия.

Допустим, у вас уже есть список «Размеры» с элементами: XS, S, M, L. Тогда в списке «Товары» вы создаете поле типа Привязка к элементам и указываете список «Размеры». Теперь, когда вы добавляете товар, вы не вводите размеры вручную, а выбираете нужные элементы из списка — например, S и M.

Чтобы получить идентификатор списка, воспользуйтесь методом [lists.get](../lists/lists-get.md). Идентификатор элемента можно узнать с помощью метода [lists.element.get](./lists-element-add.md).

**Объекты CRM.** Поле типа Привязка к элементам CRM соединяет элементы списка с объектами CRM: [лидами](../../crm/leads/index.md), [сделками](../../crm/deals/index.md), [смарт-процессами](../../crm/universal/index.md), [контактами](../../crm/contacts/index.md) и [компаниями](../../crm/companies/index.md). В элементах списка это поле отображается как ссылка на объект CRM, а в самой карточке CRM появляется вкладка с названием списка.

Формат значения поля зависит от настроек.

-  Если в настройках поля разрешена привязка только к одному объекту, то заполняйте значение поля [идентификатором объекта](../../crm/data-types.md#object_type) без символьного префикса. Например, `3` — контакт. Иначе поле будет отображаться, но не выгружаться в Excel.

-  Если разрешена привязка к нескольким объектам, то заполняйте значение поля [идентификатором с префиксом](../../crm/data-types.md#object_type). Например, `C_3` — контакт.

**Пользователи.** Для работы с пользователями в элементах списка используются поля типа Привязка к сотруднику, Кем создан и Кем изменен. Значения в полях Кем создан и Кем изменен заполняются системой автоматически. Для полей типа Привязка к сотруднику данные необходимо передавать вручную.

Идентификатор пользователя можно получить методом [user.get](../../user/user-get.md).

**Диск.** Поле типа Файл (Диск) связывает элементы списка с Диском. Чтобы добавить файл в поле элемента:

-  загрузите файл на Диск методами [disk.folder.uploadfile](../../disk/folder/disk-folder-upload-file.md) или [disk.storage.uploadfile](../../disk/storage/disk-storage-upload-file.md),

-  получите `ID` загруженного файла,

-  прикрепите файл методами [lists.element.add](./lists-element-add.md) или [lists.element.update](./lists-element-update.md).
