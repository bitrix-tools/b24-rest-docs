# Управление пользователями: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Раздел описывает работу с SIP-настройками пользователей:

- как посмотреть текущую конфигурацию сотрудника
- как включить признак SIP-аппарата

В ответе доступны основные параметры для подключения и звонков, включая линию по умолчанию, статус аппарата, сервер, учетные данные и внутренний номер.

Вызов методов зависит от права `Настройки пользователя — Изменение`. Если прав недостаточно, изменение SIP-настроек пользователя будет недоступно.

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Пользователи телефонии: базовые настройки для сотрудников](https://helpdesk.bitrix24.ru/open/18298528/)

## Связь с другими объектами

**Пользователь.** Методы раздела работают с пользователями по `USER_ID`. Идентификатор пользователя можно получить методом [user.get](../../../user/user-get.md).

{% note tip "Пользовательская документация" %}

- [Как настроить права доступа в телефонии](https://helpdesk.bitrix24.ru/open/18177766/)

{% endnote %}

## Как начать работу

1. Получите `USER_ID` сотрудника через [user.get](../../../user/user-get.md)
2. Проверьте текущие SIP-настройки методом [voximplant.user.get](./voximplant-user-get.md)
3. Включите SIP-аппарат методом [voximplant.user.activatePhone](./voximplant-user-activate-phone.md)
