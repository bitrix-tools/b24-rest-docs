# Элементы хранилища данных: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Элементы хранилища данных хранят информацию приложения. В элементах можно сохранять основные поля, значения свойств и привязку к разделам.

Группа методов `entity.item.*` позволяет создавать элементы, получать их список, менять параметры и удалять ненужные.

> Быстрый переход: [все методы](#all-methods)

{% note info "" %}

Методы раздела работают только в контексте [приложения](../../../settings/app-installation/index.md).

{% endnote %}

## Как начать работу

1. Получите идентификатор хранилища методом [entity.get](../entities/entity-get.md)

2. Если в хранилище используются разделы, получите идентификатор раздела методом [entity.section.get](../sections/entity-section-get.md)

3. Если для элементов нужны дополнительные поля, создайте свойства или получите список существующих свойств и их коды методами [entity.item.property.*](./properties/index.md)

4. Создайте первый элемент методом [entity.item.add](./entity-item-add.md)

5. Получите список элементов и их идентификаторы методом [entity.item.get](./entity-item-get.md)

6. Измените параметры элемента методом [entity.item.update](./entity-item-update.md)

7. Удалите ненужный элемент методом [entity.item.delete](./entity-item-delete.md)

## Связь с другими объектами

**Хранилище данных.** Каждый метод группы работает с конкретным хранилищем данных приложения. Для работы с хранилищами используйте методы [entity.*](../entities/index.md).

**Разделы хранилища.** Элемент можно привязать к разделу через параметр `SECTION`. Для работы с разделами используйте методы [entity.section.*](../sections/index.md).

**Свойства элементов.** Дополнительные данные элемента передаются в параметре `PROPERTY_VALUES`. Для работы со свойствами используйте методы [entity.item.property.*](./properties/index.md).
