# Разделы хранилища данных: обзор методов

Разделы хранилища данных помогают группировать элементы внутри приложения и строить иерархию.

Группа методов `entity.section.*` позволяет создавать разделы, получать их список, менять параметры и удалять ненужные.

> Быстрый переход: [все методы](#all-methods)

{% note info "" %}

Методы раздела работают только в контексте [приложения](../../../settings/app-installation/index.md).

{% endnote %}

## Как начать работу

1. Получите идентификатор хранилища методом [entity.get](../entities/entity-get.md)
2. Если в хранилище нужна иерархия, создайте первый раздел методом [entity.section.add](./entity-section-add.md)
3. Получите список разделов и их идентификаторы методом [entity.section.get](./entity-section-get.md)
4. Измените параметры раздела методом [entity.section.update](./entity-section-update.md)
5. Удалите ненужный раздел методом [entity.section.delete](./entity-section-delete.md)

## Связь с другими объектами

**Хранилище данных.** Каждый метод группы работает с конкретным хранилищем данных приложения. Для этого в методы передается параметр `ENTITY` — идентификатор хранилища. Получить его можно методом [entity.get](../entities/entity-get.md).

**Элементы хранилища.** Разделы используются для группировки элементов. Для работы с элементами используйте методы [entity.item.*](../items/index.md).

**Родительский раздел.** В методах [entity.section.add](./entity-section-add.md) и [entity.section.update](./entity-section-update.md) можно указать параметр `SECTION`, чтобы создать вложенный раздел или перенести раздел внутрь другого.
