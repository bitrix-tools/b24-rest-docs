# Хранилища данных: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Хранилища данных помогают приложениям хранить собственную информацию в Битрикс24. В хранилище можно сохранять элементы, группировать их по разделам и настраивать права доступа.

Группа методов `entity.*` позволяет создавать хранилища, получать их список, менять параметры, управлять правами и удалять ненужные.

> Быстрый переход: [все методы](#all-methods)

{% note info "" %}

Методы раздела работают только в контексте [приложения](../../../settings/app-installation/index.md).

{% endnote %}

## Как начать работу

1. Создайте новое хранилище методом [entity.add](./entity-add.md)

2. Получите список хранилищ и их идентификаторы методом [entity.get](./entity-get.md)

3. Измените параметры хранилища методом [entity.update](./entity-update.md)

4. При необходимости посмотрите или измените права доступа к хранилищу методом [entity.rights](./entity-rights.md)

5. Удалите ненужное хранилище методом [entity.delete](./entity-delete.md)

## Как работать с правами доступа

Права доступа можно задать при создании хранилища, изменить вместе с его параметрами или обновить отдельно.

#|
|| **Сценарий** | **Что использовать** ||
|| Создать хранилище сразу с нужными правами | Параметр `ACCESS` в [entity.add](./entity-add.md) ||
|| Изменить параметры хранилища и права доступа за один вызов | Параметр `ACCESS` в [entity.update](./entity-update.md) ||
|| Получить или изменить только права доступа | Метод [entity.rights](./entity-rights.md) ||
|#


## Связь с другими объектами

**Разделы хранилища.** После создания хранилища можно группировать элементы по разделам. Для работы с разделами используйте методы [entity.section.\*](../sections/index.md).

**Элементы хранилища.** Основные данные приложения хранятся в элементах. Для работы с ними используйте методы [entity.item.\*](../items/index.md).

**Свойства элементов.** Для элементов хранилища можно создавать дополнительные поля. Для работы со свойствами используйте методы [entity.item.property.\*](../items/properties/index.md).
