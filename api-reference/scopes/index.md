Скоупы определяют, к каким группам методов получит доступ приложение. Скоуп обозначается коротким кодом, например `crm`, `telephony` или `user`. Когда вы добавляете тиражное решение в партнерском кабинете или локальное решение в Битрикс24, укажите скоупы, которые нужны приложению для работы.

> Быстрый переход: [обзор страниц](#pages)
>
> Пользовательская документация: [Разработчикам: как создать вебхуки и приложения для Битрикс24](https://helpdesk.bitrix24.ru/open/20886106/)

## Как понять, какие скоупы нужны для работы приложения

Начинайте не со списка всех скоупов, а со сценария приложения. Определите, какие методы приложение будет вызывать напрямую, и проверьте блок `Scope` на страницах этих методов.

Например, приложение подключает внешнюю телефонию и напрямую вызывает методы [telephony.externalCall.register](../telephony/telephony-external-call-register.md) и [telephony.externalCall.finish](../telephony/telephony-external-call-finish.md). Так как приложение вызывает методы телефонии, в его настройках нужен скоуп `telephony`.

Скоуп `crm` для этих действий не нужен, хотя методы телефонии могут затрагивать CRM внутри Битрикс24: [telephony.externalCall.register](../telephony/telephony-external-call-register.md) может автоматически создать объект CRM, например лид, а [telephony.externalCall.finish](../telephony/telephony-external-call-finish.md) сохраняет звонок в CRM-деле.

Скоуп `crm` нужен, только если приложение само отправляет запросы к методам CRM, например [crm.lead.add](../crm/leads/crm-lead-add.md) или [crm.activity.add](../crm/timeline/activities/activity-base/crm-activity-add.md).

Порядок выбора скоупов:

1. Найдите страницы методов, которые нужны для сценария
2. Посмотрите код в блоке `Scope` в начале каждой страницы метода
3. Добавьте в настройки приложения только скоупы методов, которые приложение вызывает напрямую
4. Проверьте строку `Кто может выполнять метод`
5. Если метод требует подтверждения администратора, используйте сценарий из статьи [Вызов методов с подтверждением](./confirmation.md)

## Чем скоуп отличается от прав пользователя

Скоуп и права пользователя проверяются отдельно.

Скоуп показывает, разрешено ли приложению обращаться к группе методов. Права пользователя показывают, может ли конкретный пользователь выполнить действие в Битрикс24.

В начале страницы каждого метода есть блок:

> Scope: `crm`
>
> Кто может выполнять метод: любой пользователь

Если в строке `Кто может выполнять метод` указаны административные права или специальное право инструмента, одного скоупа недостаточно. Метод должен выполняться от имени пользователя, у которого есть нужные права.
