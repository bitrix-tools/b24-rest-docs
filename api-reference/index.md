# Справочник REST API: инструменты и методы Битрикс24

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

С помощью REST API приложение работает с инструментами Битрикс24. Например, оно может создать сделку в CRM, получить задачу, загрузить файл на Диск, обработать событие об изменении данных или встроить виджет в карточку.

> Быстрый переход: [все разделы справочника](#all-methods)

## Как пользоваться справочником

Начните с того, что уже известно: какие данные приложение должно получать или изменять, какой метод нужно вызвать или какие права выдать.

Например, для работы с клиентами откройте [CRM](./crm/index.md). Там собраны методы для лидов, сделок, контактов, компаний и других объектов CRM. Для задач подойдет раздел [Задачи](./tasks/index.md), для файлов — [Диск](./disk/index.md), для заказов интернет-магазина — [Интернет-магазин](./sale/index.md).

Если знаете название метода, ищите его в профильном разделе или через поиск. По названию многих методов можно понять, с каким объектом работает метод и какое действие выполняет: [crm.deal.add](./crm/deals/crm-deal-add.md) создает сделку, [tasks.task.get](./tasks/tasks-task-get.md) получает задачу, [disk.file.get](./disk/file/disk-file-get.md) получает файл.

Если название нужного метода неизвестно, откройте [обзор разделов справочника](#all-methods) и выберите подходящий раздел. На странице раздела есть список методов и ссылки на связанные материалы.

Права приложения проверяйте в разделе [Скоупы методов](./scopes/index.md). Скоуп определяет, к каким группам методов приложение может обращаться через REST API. Его указывают при настройке прав приложения.

Скоуп не открывает все данные сам по себе. Приложение увидит только те данные, к которым есть доступ у пользователя, от имени которого выполняется запрос.

## Какая информация есть на страницах методов

Страницы методов помогают подготовить отдельные запросы. В них указано, какое действие выполняет метод, какие параметры передать и какой ответ вернет сервер.

Они не описывают рабочий сценарий целиком. Если нужно решить задачу полностью, начните с обзорной страницы инструмента или посмотрите [Туториалы](../tutorials/index.md).
