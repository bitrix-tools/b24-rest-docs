# Роботы приложений: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Роботы приложений добавляют в автоматизацию Битрикс24 шаг, который вызывает обработчик приложения. Такой шаг принимает входные параметры, может дождаться ответа внешней системы и вернуть выходные значения в процесс.

В REST API роботы приложений и [действия приложений](../bizproc-activity/index.md) близки по модели. Для каждого типа есть отдельные методы регистрации, обновления, удаления и получения списка. Внутри платформы используется общий механизм выполнения.

Роботов можно использовать в CRM-автоматизации, бизнес-процессах и умных сценариях. Поэтому для новой разработки рекомендуется выбирать роботов, а действия приложений использовать в первую очередь для поддержки уже существующих интеграций.

## Какие задачи решают роботы приложений

- Выполняют внешние действия в автоматизации CRM и бизнес-процессах.
- Возвращают вычисленные данные в процесс для следующих шагов.
- Ограничивают доступность по типу документа и редакции Битрикс24.
- Выводят сообщения в лог в ходе выполнения сценария.

## Как работает ожидание результата

Если робот должен дождаться ответа внешней системы, укажите `RETURN_PROPERTIES` и `USE_SUBSCRIPTION = 'Y'` при регистрации или обновлении через [bizproc.robot.add](./bizproc-robot-add.md) и [bizproc.robot.update](./bizproc-robot-update.md).

Дальше сценарий работает так:

1. Робот запускается в автоматизации и отправляет данные на обработчик приложения `HANDLER`.
2. Приложение получает уникальный ключ `EVENT_TOKEN` и рассчитывает результат.
3. Приложение возвращает значения в процесс с помощью метода [bizproc.event.send](./bizproc-event-send.md).
4. При необходимости приложение добавляет пояснение в журнал процесса через `LOG_MESSAGE`.

## Как начать работу

1. Зарегистрируйте робота методом [bizproc.robot.add](./bizproc-robot-add.md).
2. Разместите робота в нужной автоматизации CRM или бизнес-процессе и запустите сценарий.
3. Обновите настройки робота методом [bizproc.robot.update](./bizproc-robot-update.md).
4. Проверьте список зарегистрированных роботов через [bizproc.robot.list](./bizproc-robot-list.md).
5. Удалите неактуальных роботов с помощью метода [bizproc.robot.delete](./bizproc-robot-delete.md).

## Что важно учитывать

- Методы управления роботами [bizproc.robot.add](./bizproc-robot-add.md), [bizproc.robot.update](./bizproc-robot-update.md), [bizproc.robot.list](./bizproc-robot-list.md), [bizproc.robot.delete](./bizproc-robot-delete.md) работают только в контексте установленного приложения.
- Для [bizproc.event.send](./bizproc-event-send.md) нужен валидный `EVENT_TOKEN`, иначе метод вернет ошибку доступа.

## Связь с другими объектами

- **CRM-объекты.** Через `DOCUMENT_TYPE` и `FILTER` робот связывается с лидами, сделками и другими типами документов, где он должен быть доступен.
- **Действия приложений.** Роботы и действия используют общий внутренний механизм, поэтому при проектировании интеграции важно заранее выбрать основной путь для новых сценариев.
