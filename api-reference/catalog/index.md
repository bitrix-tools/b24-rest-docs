# Торговый каталог: обзор разделов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Торговый каталог в Битрикс24 — это единый каталог товаров и услуг для CRM и Интернет-магазина. Он помогает хранить актуальные данные по позициям, быстрее собирать документы в CRM и автоматизировать обмен с внешними системами.

Разделы Торгового каталога охватывают основные данные и настройки:

- товары, услуги и вариации
- цены, типы цен, наценки и правила округления
- свойства товаров и их значения
- структура каталога и изображения товаров
- единицы измерения, коэффициенты и ставки НДС
- склады, остатки и документы складского учета

> Быстрый переход: [все разделы каталога](#all-methods)
>
> Пользовательская документация: [Каталог товаров и услуг: что это и как с ним работать](https://helpdesk.bitrix24.ru/open/26307326/)

## Как начать работу

1. Проверьте структуру нужного объекта, названия полей и их типы на странице [Типы данных и структура объектов в REST API Каталога](./data-types.md).
2. Определите основной объект сценария: товар, цена, свойство, склад или документ учета.
3. Откройте профильный раздел и получите список объектов методом `list`, чтобы собрать рабочие идентификаторы.
4. Выполните изменение объекта методами `add` или `update` нужного раздела.
5. Проверьте результат методом `get` и при необходимости уточните доступные поля методом `getFields`.

## Связь каталога с другими объектами

**CRM.** Данные каталога используются в товарных позициях сделок, счетов и предложений: передаются идентификатор товара, цена, скидка и НДС. Для сценариев с поставщиками используйте методы раздела [Работа с CRM-поставщиками](./documentcontractor/index.md).

**Интернет-магазин.** Каталог формирует данные для витрины и карточек товаров Интернет-магазина.

**Файлы.** В методах каталога файлы и изображения передаются в формате `fileData` или `fileContent`: имя файла и содержимое в Base64. Формат загрузки и примеры приведены на странице [Как загрузить файлы](../files/how-to-upload-files.md).
