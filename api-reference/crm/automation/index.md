# Автоматизация CRM: обзор методов

Методы раздела позволяют передавать внешние события в автоматизацию CRM через триггеры. Если для объекта настроен триггер, он переводит объект на другую стадию или в другой статус.

Раздел включает два сценария. В первом запускают готовый webhook-триггер методом [crm.automation.trigger](./crm-automation-trigger.md) — триггер настраивают в интерфейсе Битрикс24, затем вызывают метод через API. Во втором приложение регистрирует триггер и управляет им через методы раздела [Триггеры автоматизации CRM](./triggers/index.md).

## Как начать работу

**Если webhook-триггер уже настроен в Битрикс24:**

1. Возьмите значение `code` из URL триггера «Отследить входящий вебхук» в настройках автоматизации CRM
2. Составьте `target` для целевого объекта, например `DEAL_25`
3. Вызовите [crm.automation.trigger](./crm-automation-trigger.md), передав `target` и `code`
4. При необходимости проверьте результат методом [crm.item.get](../universal/crm-item-get.md)

**Если нужно управлять триггерами из приложения:**

Полный порядок действий — в разделе [Триггеры автоматизации CRM](./triggers/index.md): регистрация, привязка к стадии и запуск.

## Связь с другими объектами

**Автоматизация CRM.** В обоих сценариях триггер нужно заранее привязать к стадии или статусу в настройках автоматизации Битрикс24.

**Объекты CRM.** Триггер запускают для конкретного объекта CRM. В webhook-сценарии в параметре `target` передают строковый идентификатор объекта в формате [`TYPENAME_ID`](../../data-types.md), например `DEAL_25`, где `DEAL` — тип объекта CRM, `25` — его числовой идентификатор из [crm.item.list](../universal/crm-item-list.md). В сценарии приложения тип объекта `OWNER_TYPE_ID` получают методом [crm.enum.ownertype](../auxiliary/enum/crm-enum-owner-type.md), идентификатор объекта `OWNER_ID` — методом [crm.item.list](../universal/crm-item-list.md).
