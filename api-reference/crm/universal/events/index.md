События дают возможность приложениям реагировать на изменения практически в реальном времени: получать уведомления о создании, обновлении или удалении элементов смарт-процессов.

Подробно работа с событиями описана в статье [Концепция и преимущества обработки событий](../../../events/index.md).

> Быстрый переход: [все события](#all-events)

## Как получать события

Подписаться на события элементов смарт-процессов можно через:

- [исходящий вебхук](../../../../local-integrations/local-webhooks.md)
- [приложение](../../../../settings/app-installation/index.md) и метод [event.bind](../../../events/event-bind.md)

Пример кода обработчика для события описан в статье [Как проверить свой обработчик для обработки событий Битрикс24](../../../events/test-handler.md).

## Как выбрать, какие события получать

Есть два способа получать события смарт-процессов: подписаться на базовые события для всех смарт-процессов или на событие одного определенного смарт-процесса.

**Базовое событие для всех смарт-процессов.** Если обработчик должен получать события по элементам всех смарт-процессов, подпишитесь на события:

- [onCrmDynamicItemAdd](./on-crm-dynamic-item-add.md) — создание элемента смарт-процесса
- [onCrmDynamicItemUpdate](./on-crm-dynamic-item-update.md) — обновление элемента смарт-процесса
- [onCrmDynamicItemDelete](./on-crm-dynamic-item-delete.md) — удаление элемента смарт-процесса

Базовое событие срабатывает для элементов разных смарт-процессов. В обработчик приходит объект `data.FIELDS`:

- `ID` — идентификатор элемента смарт-процесса
- `ENTITY_TYPE_ID` — идентификатор смарт-процесса, которому принадлежит элемент

Чтобы обработать события нужного смарт-процесса, проверяйте значение `data.FIELDS.ENTITY_TYPE_ID`. Идентификатор `entityTypeId` смарт-процесса можно получить методом [crm.type.list](../user-defined-object-types/crm-type-list.md).

**Событие для одного смарт-процесса.** Если обработчик должен получать события только определенного смарт-процесса, используйте событие с суффиксом `_{entityTypeId}`. Такой суффикс указывает, к какому смарт-процессу относится событие.

Например, для смарт-процесса с `entityTypeId = 147` событие создания элемента будет называться `ONCRMDYNAMICITEMADD_147`.

Перед подпиской проверьте, доступно ли такое событие:

- для приложения — методом [events](../../../events/events.md)
- для исходящего вебхука — в конструкторе [исходящих вебхуков](../../../../local-integrations/local-webhooks.md)

Если события с суффиксом нет, подпишитесь на базовое событие [onCrmDynamicItemAdd](./on-crm-dynamic-item-add.md), [onCrmDynamicItemUpdate](./on-crm-dynamic-item-update.md) или [onCrmDynamicItemDelete](./on-crm-dynamic-item-delete.md). В обработчике проверьте поле `data.FIELDS.ENTITY_TYPE_ID`: в нем приходит `entityTypeId` смарт-процесса, к которому относится элемент.
