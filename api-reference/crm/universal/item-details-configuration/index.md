# Разделы в детальной карточке CRM: обзор методов

{% note tip "" %}

Разделы в детальной карточке CRM отображают информацию об элементе. Например, в карточке сделки могут быть разделы о клиенте, заказе и условиях доставки.

Методы `crm.item.details.configuration.*` управляют разделами карточки. Можно сохранять личные настройки пользователя или задавать единый вид карточки для всей компании.

> Быстрый переход: [все методы](#all-methods)

## Как работать с настройками карточки

1. Определите тип объекта CRM через `entityTypeId`. Идентификаторы смотрите в разделах [Системные типы CRM](../../index.md) и [Смарт-процессы](../user-defined-object-types/index.md).
2. Для сделок и смарт-процессов укажите воронку в параметре `extras` (`dealCategoryId` или `categoryId`). Получить список воронок можно методом [crm.category.list](../category/crm-category-list.md).
3. Получите текущую конфигурацию методом [crm.item.details.configuration.get](./crm-item-details-configuration-get.md).
4. Сохраните изменения методом [crm.item.details.configuration.set](./crm-item-details-configuration-set.md) или сбросьте настройки к значениям по умолчанию через [crm.item.details.configuration.reset](./crm-item-details-configuration-reset.md).
5. Чтобы сделать общую конфигурацию обязательной для всех пользователей, используйте [crm.item.details.configuration.forceCommonScopeForAll](./crm-item-details-configuration-forceCommonScopeForAll.md).

## Область действия настроек

Параметр `scope` определяет, чьи настройки будут изменены.

#|
|| **Код scope** | **Назначение** ||
|| `P` | Личная конфигурация конкретного пользователя. Используется по умолчанию ||
|| `C` | Общая конфигурация для всех пользователей Битрикс24 ||
|#

Для работы с личными настройками достаточно прав обычного пользователя. Изменение общих или чужих настроек требует прав администратора.

## Связь с другими объектами

**Тип объекта CRM.** Настройки карточки применяются к конкретному типу объекта через параметр `entityTypeId`. Для системных объектов используйте идентификаторы из раздела [Системные типы CRM](../../index.md), для смарт-процессов — из раздела [Смарт-процессы](../user-defined-object-types/index.md).

**Воронки.** Для сделок и смарт-процессов вид карточки может отличаться в разных воронках. Указывайте идентификатор воронки в параметре `extras`. Если параметр не передан, система использует воронку по умолчанию.

**Пользователь.** Личные настройки карточки привязаны к пользователю через параметр `userId` при `scope = 'P'`. Если `userId` не передан, методы используют пользователя, который выполняет запрос. Получить идентификатор пользователя можно методом [user.get](../../../user/user-get.md).

## Частые причины ошибок

- Указан неподдерживаемый тип объекта CRM.
- Переданы некорректные параметры `extras` для выбранного типа.
- Недостаточно прав для изменения общей конфигурации.
