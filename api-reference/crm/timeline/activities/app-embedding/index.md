# Встраивание приложений в дела CRM

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

Дело CRM — это запись о работе с клиентом: звонок, встреча, письмо или задача. Приложение можно встроить в дела CRM. Сотрудник сможет запускать действия внешнего сервиса из карточки лида, сделки, коммерческого предложения, нового счета или смарт-процесса. Отдельную программу открывать не придется.

> Быстрый переход: [сценарии и точки встраивания](#all-placements)
>
> Пользовательская документация: [Таймлайн в карточке CRM](https://helpdesk.bitrix24.ru/open/23960160/)

## Какие сценарии доступны

Приложение может работать с делами CRM двумя способами:

**Действие для существующего дела.** Приложение добавляет пункт в контекстное меню дела. После выбора пункта Битрикс24 открывает обработчик приложения. Для этого сценария есть две точки встраивания: пункт меню в списке дел и пункт меню в таймлайне карточки CRM. Пункты меню подключаются через [механизм виджетов](../../../../widgets/index.md).

**Дело, созданное приложением.** Приложение создает дело методом [crm.activity.add](../activity-base/crm-activity-add.md) с провайдером `REST_APP`. `REST_APP` — фиксированное значение поля `PROVIDER_ID`, которое показывает, что дело создано приложением. По клику на такое дело Битрикс24 открывает приложение.

Чтобы приложение открылось внутри Битрикс24, ему нужна своя веб-страница — обработчик. Страница должна быть доступна по внешнему URL, чтобы Битрикс24 мог ее открыть.

## Как начать работу

1. Подготовьте обработчик — страницу приложения с URL, доступным из внешней сети
2. Выберите сценарий и точку встраивания в таблице [Как выбрать сценарий и точку встраивания](#all-placements)

**Если приложение добавляет пункт в меню дела.** Зарегистрируйте обработчик методом [placement.bind](../../../../widgets/placement-bind.md) и передайте нужный код места встраивания в `PLACEMENT`. Пример запроса смотрите в блоке [Примеры кода](../../../../widgets/placement-bind.md#примеры-кода) метода [placement.bind](../../../../widgets/placement-bind.md). После регистрации проверьте, что пункт меню появился в списке дел CRM или в таймлайне карточки CRM.

**Если приложение создает свое дело в CRM.** Используйте метод [crm.activity.add](../activity-base/crm-activity-add.md) в установленном приложении и передайте в `fields` значение `PROVIDER_ID=REST_APP`. Так Битрикс24 поймет, что дело принадлежит приложению, и при открытии покажет страницу приложения в слайдере.

Дела с провайдером `REST_APP` можно создать, изменить или удалить только из приложения. При вызове методов [crm.activity.add](../activity-base/crm-activity-add.md), [crm.activity.update](../activity-base/crm-activity-update.md) или [crm.activity.delete](../activity-base/crm-activity-delete.md) через вебхук Битрикс24 вернет ошибку `Application context required`.

В `PROVIDER_TYPE_ID` можно передать тип дела приложения, например `LINK`. Тип помогает приложению различать виды созданных дел. Если параметр не передан, Битрикс24 использует `LINK` по умолчанию. После создания откройте дело и проверьте, что Битрикс24 открывает страницу приложения в слайдере.

## Связь с другими объектами

**Дела CRM.** Встраивание работает с делом как с объектом CRM: к существующему делу добавляется пункт меню или приложение создает собственное дело. Для работы с делами используйте методы [crm.activity.add](../activity-base/crm-activity-add.md), [crm.activity.get](../activity-base/crm-activity-get.md), [crm.activity.update](../activity-base/crm-activity-update.md) и [crm.activity.delete](../activity-base/crm-activity-delete.md).

**Карточка CRM.** Точки встраивания таймлайна связывают действие приложения с карточкой CRM, в которой открыто дело.

## Как выбрать сценарий и точку встраивания {#all-placements}

Когда Битрикс24 открывает обработчик, он передает странице данные вызова в `PLACEMENT_OPTIONS`. Это набор параметров, по которым приложение понимает, откуда его открыли и с каким делом нужно работать. Состав `PLACEMENT_OPTIONS` зависит от выбранной точки встраивания.

Код `CRM_XXX_ACTIVITY_TIMELINE_MENU` в таблице — это шаблон для точек встраивания в таймлайне карточек CRM. Вместо `XXX` используется тип карточки. 
Полный список кодов можно посмотреть на странице [Пункт контекстного меню дела в карточке элемента](../../../../widgets/crm/activity-timeline-menu.md).
