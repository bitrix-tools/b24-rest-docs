# Дополнительные контентные блоки таймлайна: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Дополнительные контентные блоки — это настраиваемые элементы интерфейса в записи таймлайна CRM, которые приложение добавляет рядом с основным содержимым записи.

Дополнительные блоки нужны, когда стандартного вида записи недостаточно и нужно отобразить данные приложения прямо в карточке элемента CRM без перехода в отдельный интерфейс. С помощью таких блоков можно показать в таймлайне поясняющий текст, ссылку, заголовок с вложенным блоком и другие элементы.

> Быстрый переход: [все методы](#all-methods)

## Что учитывать перед вызовом методов

- Методы работают только в контексте [приложения](../../../../settings/app-installation/index.md).
- Приложение может получать и удалять только тот набор блоков, который установило само.
- При повторном вызове [crm.timeline.layout.blocks.set](./crm-timeline-layout-blocks-set.md) предыдущий набор блоков этого же приложения перезаписывается.

## Ограничения применения

- Наборы дополнительных контентных блоков нельзя установить для дел. Для дел используйте методы [crm.activity.layout.blocks.*](../activities/layout-blocks/index.md).
- Наборы дополнительных контентных блоков нельзя установить для [лог-записей таймлайна](../logmessage/index.md).
- Наборы дополнительных контентных блоков нельзя установить для устаревших записей таймлайна.

## Жизненный цикл наборов блоков

- При удалении записи таймлайна все наборы дополнительных контентных блоков в этой записи удаляются безвозвратно.
- При удалении приложения удаляются все наборы дополнительных контентных блоков, которые это приложение добавило в записи таймлайна.

## Как работать с дополнительными блоками

1. Подготовьте описание набора дополнительных контентных блоков в формате [RestAppLayoutDto](../activities/configurable/structure/rest-app-layout-dto.md).
2. Установите набор блоков методом [crm.timeline.layout.blocks.set](./crm-timeline-layout-blocks-set.md).
3. Получите установленный набор методом [crm.timeline.layout.blocks.get](./crm-timeline-layout-blocks-get.md).
4. Удалите набор методом [crm.timeline.layout.blocks.delete](./crm-timeline-layout-blocks-delete.md), если он больше не нужен.
5. Если нужен готовый рабочий сценарий, посмотрите [пример тестового приложения](./content-blocks-test-app.md).
