# Информация о пользователе: обзор методов

Методы этой группы работают только с текущим пользователем: получают базовые данные профиля, проверяют коды доступа и права на управление настройками приложений.

Информацию об остальных пользователях получает группа методов [Пользователи](../../user/index.md).

> Быстрый переход: [все методы](#all-methods)

## Как получить данные текущего пользователя

Метод [profile](./profile.md) получает базовые данные текущего пользователя, например `ID`, имя, фамилию, признак администратора и часовой пояс.

## Как проверить доступ текущего пользователя

Метод [user.access](./user-access.md) проверяет наличие у текущего пользователя хотя бы одного из переданных кодов доступа, например `G2` или `AU`.

Метод [user.admin](./user-admin.md) проверяет наличие у текущего пользователя прав на управление настройками приложений.

## Коды доступа в user.access   {#kody-dostupa-v-useraccess}

Метод [user.access](./user-access.md) принимает массив кодов доступа в параметре `ACCESS`.

#|
|| **Код** | **Что означает** | **Пример** ||
|| `U<id>` | Пользователь | `U1` ||
|| `G<id>` | Группа пользователей | `G2` ||
|| `AU` | Все авторизованные пользователи | `AU` ||
|#


Если код доступа уже известен, используйте [user.access](./user-access.md). Чтобы получить название кода доступа, используйте [access.name](../system/access-name.md).
