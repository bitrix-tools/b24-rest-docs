# Обзор событий при работе с приложением

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

События дают возможность приложениям реагировать на изменения практически в реальном времени: получать уведомления об установке, обновлении, удалении и оплате приложения, добавлении пользователя и решении администратора по запросу на использование методов с подтверждением.

Подробно работа с событиями описана в статье [Концепция и преимущества обработки событий](../../events/index.md).

> Быстрый переход: [все события](#all-events)

{% note info "" %}

События раздела работают только в контексте [приложения](../../../settings/app-installation/index.md).

{% endnote %}

## Как получать события

Подписаться на события приложения можно через [приложение](../../../settings/app-installation/index.md) и метод [event.bind](../../events/event-bind.md).

Чтобы получать события:

- установите приложение и укажите публичный URL обработчика
- если приложение запрашивает доступ к методам с подтверждением, обработайте событие [onAppMethodConfirm](./on-app-method-confirm.md)

Пример кода обработчика для события описан в статье [Как проверить свой обработчик для обработки событий Битрикс24](../../events/test-handler.md).
