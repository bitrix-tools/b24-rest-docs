# Системные методы: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Системные методы проверяют доступность методов и scope, получают информацию о приложении, кодах доступа, функционале портала и времени сервера.

> Быстрый переход: [все методы](#all-methods)

## Что проверять перед вызовом метода

**Контекст приложения.** Метод [app.info](./app-info.md) работает только в контексте приложения. Иначе метод вернет ошибку `ACCESS_DENIED`.

**Доступность метода.** Для новых интеграций используйте [method.get](./method-get.md). Метод показывает, есть ли он на портале и доступен ли вызов с текущими правами.

**Функционал портала.** Метод [feature.get](./feature-get.md) нужен, когда поведение зависит от scope и функционала портала.

## Связь с другими объектами

**Права доступа.** Метод [access.name](./access-name.md) расшифровывает коды `ACCESS`, которые использует, например, [user.access](../users/user-access.md). Права пользователя, а не приложения, проверяет [user.access](../users/user-access.md).

**Разрешения приложения.** Метод [scope](./scope.md) возвращает scope текущего приложения. Значения кодов scope описаны на странице [доступных scope](../../scopes/permissions.md).

**Пользователь.** Метод [profile](../users/profile.md) получает базовые данные текущего пользователя. Методы [user.access](../users/user-access.md) и [user.admin](../users/user-admin.md) проверяют коды доступа и права пользователя.
