# Общие методы и события: обзор

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Общие методы и события помогают проверить контекст приложения, получить данные текущего пользователя, сохранить настройки интеграции и обработать жизненный цикл приложения.

Для методов раздела не нужны отдельные scope. Дополнительные ограничения зависят от контекста приложения и прав текущего пользователя.

## Что учитывать перед началом работы

**Контекст приложения.** Методы `app.*`, `user.option.*`, события раздела и часть системных методов работают только в контексте [приложения](../../settings/app-installation/index.md). Это важно учесть до первого вызова.

**Проверка доступности.** Перед вызовом метода проверьте его доступность через [method.get](./system/method-get.md), а список scope получите через [scope](./system/scope.md).

**Коды доступа.** Для проверки прав текущего пользователя используйте [user.access](./users/user-access.md). Расшифровать коды `ACCESS` помогает [access.name](./system/access-name.md).

**Настройки приложения.** Общие параметры интеграции храните через `app.option.*`, а персональные настройки пользователя — через `user.option.*`.

## Связь с другими объектами

Раздел связан с приложением, текущим пользователем и разрешениями доступа.

**Приложение.** Методы [app.info](./system/app-info.md), [app.option.get](./settings/app-option-get.md) и [app.option.set](./settings/app-option-set.md) работают с данными и настройками текущего приложения.

**Пользователь.** Метод [profile](./users/profile.md) получает ID, имя, фамилию и другие данные текущего пользователя. Методы [user.admin](./users/user-admin.md) и [user.access](./users/user-access.md) проверяют его права перед вызовом методов с ограничениями.

**Разрешения.** Метод [scope](./system/scope.md) возвращает scope текущего приложения. Значения кодов scope описаны на странице [доступных scope](../scopes/permissions.md).
