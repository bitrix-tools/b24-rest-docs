# Настройки приложения: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Настройки приложения хранят общие параметры интеграции и персональные значения для каждого пользователя.

> Быстрый переход: [все методы](#all-methods)

{% note info "" %}

Методы раздела работают только в контексте [приложения](../../../settings/app-installation/index.md).

{% endnote %}

## Как выбрать группу методов

Если настройка одна для всего приложения, используйте группу методов `app.option.*`. Она подходит для общих параметров интеграции, которые не зависят от пользователя.

Если у каждого пользователя должны быть свои значения, используйте группу методов `user.option.*`. Например, для персональных настроек интерфейса или рабочих параметров.

## Что учитывать перед вызовом методов

**Права доступа.** Метод [app.option.set](./app-option-set.md) доступен только администратору. Методы чтения настроек и `user.option.*` доступны любому пользователю в контексте приложения.

**Привязка к пользователю.** Методы [user.option.set](./user-option-set.md) и [user.option.get](./user-option-get.md) сохраняют и читают не общие настройки приложения, а настройки конкретного пользователя. Используйте их, когда у разных пользователей должны быть разные значения, например для персональных настроек интерфейса.

## Связь с другими объектами

**Приложение.** Группа методов [Системные методы](../system/index.md) проверяет контекст приложения, доступные scope и наличие метода на портале. Например, [app.info](../system/app-info.md) получает информацию о приложении, а [scope](../system/scope.md) возвращает список доступных scope.

**Пользователь.** Метод [user.admin](../users/user-admin.md) проверяет права текущего пользователя перед сохранением настроек. Базовые данные текущего пользователя получает [profile](../users/profile.md).
