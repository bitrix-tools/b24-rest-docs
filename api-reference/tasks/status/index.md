# Смена статуса задачи: обзор методов

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

Методы раздела переводят задачу между основными рабочими статусами: позволяют начать выполнение, приостановить работу, отложить задачу, завершить ее и снова вернуть в работу после завершения.

Статус задачи отражает текущий этап работы над ней и влияет на карточку задачи, историю изменений и доступные действия участников.

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Как контролировать задачи сотрудников в Битрикс24](https://helpdesk.bitrix24.ru/open/27774804/)

## Как начать работу

1. Создайте задачу методом [tasks.task.add](../tasks-task-add.md) или получите ее идентификатор методом [tasks.task.list](../tasks-task-list.md)
2. Получите текущие данные задачи методом [tasks.task.get](../tasks-task-get.md), если нужно проверить текущий статус и доступные действия
3. Переведите задачу в работу методом [tasks.task.start](./tasks-task-start.md), когда исполнитель начинает выполнение
4. Приостановите работу методом [tasks.task.pause](./tasks-task-pause.md), если задачу нужно временно остановить
5. Отложите задачу методом [tasks.task.defer](./tasks-task-defer.md), если к ней нужно вернуться позже
6. Завершите задачу методом [tasks.task.complete](./tasks-task-complete.md), когда работа выполнена
7. Возобновите задачу методом [tasks.task.renew](./tasks-task-renew.md), если после завершения ее нужно вернуть в работу

## Связь с другими объектами

**Задача.** Все методы раздела работают с существующей задачей по ее идентификатору. Создать задачу можно методом [tasks.task.add](../tasks-task-add.md), получить данные по задаче — методом [tasks.task.get](../tasks-task-get.md), получить список задач — методом [tasks.task.list](../tasks-task-list.md).

**Пользователи.** Возможность смены статуса зависит от роли пользователя в задаче и его прав доступа. Проверить доступ к задаче можно методом [tasks.task.getaccess](../tasks-task-get-access.md).

**История задачи.** Изменение статуса фиксируется в истории задачи. Получить историю изменений можно методом [tasks.task.history.list](../tasks-task-history-list.md).

**Контроль выполнения.** Если в задаче включен контроль выполнения, после перевода задачи в статус «завершена» постановщик может принять результат методом [tasks.task.approve](../tasks-task-approve.md) или вернуть задачу в работу методом [tasks.task.disapprove](../tasks-task-disapprove.md).
