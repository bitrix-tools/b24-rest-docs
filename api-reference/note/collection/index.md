# Базы знаний 2.0: обзор методов

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

База знаний 2.0 помогает собирать документы по темам, выстраивать общую структуру и открывать доступ к материалам нужным сотрудникам. Методы `note.collection.*` управляют самой базой знаний как контейнером верхнего уровня: создают новую базу, возвращают одну или несколько баз, отдают схему полей, переименовывают, архивируют и переносят в корзину.

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Битрикс24 База знаний 2.0: как создать и настроить](https://helpdesk.bitrix24.ru/open/28444346/)

Методы раздела относятся к REST 3.0. Особенности вызова и формат ответа новой версии API описаны в [обзоре REST 3.0](../../rest-v3.md).

{% endnote %}

## Как начать работу

1. Создайте базу знаний методом [note.collection.add](./note-collection-add.md)
2. Получите список доступных баз знаний методом [note.collection.list](./note-collection-list.md), если нужно выбрать нужную базу знаний по идентификатору
3. Получите данные одной базы знаний методом [note.collection.get](./note-collection-get.md), если нужно открыть ее карточку по идентификатору
4. Уточните состав и типы полей методами [note.collection.field.list](./note-collection-field-list.md) и [note.collection.field.get](./note-collection-field-get.md), если строите форму, таблицу или валидацию на своей стороне
5. Используйте [note.collection.update](./note-collection-update.md), если нужно изменить название базы знаний
6. Архивируйте базу знаний методом [note.collection.archive](./note-collection-archive.md), если нужно скрыть ее из активной работы, но сохранить содержимое
7. Удаляйте базу знаний методом [note.collection.delete](./note-collection-delete.md), если нужно перенести ее и документы внутри нее в корзину

## Ограничения и особенности

- Методы `note.collection.*` работают только с базой знаний целиком. Для создания, изменения и удаления отдельных страниц нужны методы раздела [Документы](../document/index.md)
- Архивация базы знаний затрагивает не только сам контейнер, но и все документы внутри него. После архивации документы остаются в базе знаний, но выходят из активной работы
- Удаление базы знаний переносит в корзину и саму базу знаний, и все документы внутри нее. Если в базе знаний много материалов, удаление затронет всю эту структуру сразу
- Доступ к созданию, переименованию, архивации и удалению зависит от прав текущего пользователя. Если прав недостаточно, метод вернет ошибку доступа

## Связь с другими объектами

**Документы.** База знаний — это контейнер для документов. Сначала создают саму базу знаний, а затем наполняют ее страницами и вложенными разделами через методы раздела [Документы](../document/index.md). Все документы, которые относятся к одной теме или одному отделу, можно держать внутри одной базы знаний.
