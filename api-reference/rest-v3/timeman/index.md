Записи рабочего времени помогают контролировать, когда сотрудник начал и завершил рабочий день, сколько времени заняли перерывы и сколько часов было отработано за выбранный период. Эти данные используют, чтобы анализировать рабочую нагрузку, отслеживать незакрытые или неподтвержденные дни и сверять фактически отработанное время.

## Как начать работу

Методы `timeman.record.*` возвращают записи о рабочем времени конкретного сотрудника и структуру полей этого объекта. Для изменения данных используйте методы раздела [Учет рабочего времени](../../timeman/index.md).

1. Определите идентификатор сотрудника `userId`, для которого нужно получить записи о рабочем времени
2. Сформируйте запрос к [timeman.record.list](./timeman-record-list.md): в `filter.userId` передайте идентификатор сотрудника, а при необходимости добавьте условие по `startTime`, чтобы получить записи за нужный период
3. В этом же запросе укажите `select`, если нужно вернуть только определенные поля
4. В этом же запросе используйте `order` и `pagination`, если нужно отсортировать записи по времени начала рабочего дня и ограничить объем выборки
5. Уточните доступные поля через [timeman.record.field.list](./timeman-record-field-list.md), если нужно проверить, какие поля доступны в ответе, фильтре и сортировке
6. Получите описание конкретного поля через [timeman.record.field.get](./timeman-record-field-get.md), если нужно понять тип данных и метаданные одного поля перед обработкой ответа

## Ограничения раздела

- Метод [timeman.record.list](./timeman-record-list.md) требует обязательный фильтр по `userId`
- В одном запросе к [timeman.record.list](./timeman-record-list.md) можно передать только один идентификатор сотрудника
- Доступ к записям рабочего времени зависит от прав чтения и настроек субординации

## Связь с другими объектами

**Пользователь.** В [timeman.record.list](./timeman-record-list.md) идентификатор сотрудника передается в `filter.userId`. Получить его можно методом [user.get](../../user/user-get.md).

**Период рабочего времени.** Для отбора записей за нужный день, неделю или другой период используется поле `startTime`. Через него можно построить выборку по диапазону дат и получить только те записи, которые относятся к нужному отчетному периоду.
