# Файлы в Базе знаний 2.0: обзор методов

{% note tip "" %}

Файлы в Базе знаний 2.0 привязываются к документам и используются как вложения в Markdown: изображения, видео и обычные файлы. Методы `note.file.*` помогают загрузить файл, получить его метаданные, узнать схему полей и вернуть блок Markdown для вставки в документ.

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Битрикс24 База знаний 2.0: как создать и настроить](https://helpdesk.bitrix24.ru/open/28444346/)

## Как начать работу

1. Создайте документ методом [note.document.add](../document/note-document-add.md), если у вас еще нет страницы, к которой нужно привязать файл
2. Загрузите файл методом [note.file.add](./note-file-add.md) и сохраните возвращенный `id`
3. Запросите метаданные файла и блок Markdown методом [note.file.get](./note-file-get.md)
4. Уточните состав и типы полей методами [note.file.field.list](./note-file-field-list.md) и [note.file.field.get](./note-file-field-get.md), если строите форму, таблицу или валидацию на своей стороне

## Ограничения и особенности

- Метод `note.file.add` только загружает файл и привязывает его к документу, но не вставляет вложение в текст документа. После загрузки нужно отдельно получить блок Markdown и обновить страницу
- Лимит размера файла берется из настройки `main.max_file_size`, а если она не задана, используется ограничение `25 МиБ (25 * 1024 * 1024 байт)`. Поэтому допустимый размер может отличаться в разных Битрикс24
- Разрешены только определенные расширения файлов. Если расширение не входит в список допустимых, файл не будет загружен

## Связь с другими объектами

**Документы.** Файл всегда относится к конкретной странице, а не к базе знаний в целом. Поэтому перед загрузкой файла сначала создают документ или выбирают уже существующий документ, в котором должно появиться вложение. Сценарий состоит из трех шагов:
- загрузить файл методом [note.file.add](./note-file-add.md)
- получить блок `assetMarkdown` методом [note.file.get](./note-file-get.md)
- вставить этот блок в `markdown` и обновить документ методом [note.document.update](../document/note-document-update.md)
