# Документы в Базе знаний 2.0: обзор методов

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

Документы в Базе знаний 2.0 хранят содержимое базы знаний: заголовки, вложенные страницы и текст в Markdown. Методы `note.document.*` помогают создать документ, обновить его заголовок или содержимое, получить схему полей документов, дерева и поиска, архивировать ветку документов и перенести ее в корзину.

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Битрикс24 База знаний 2.0: как создать и настроить](https://helpdesk.bitrix24.ru/open/28444346/)

## Как начать работу

1. Создайте базу знаний методом [note.collection.add](../collection/note-collection-add.md), если ее еще нет
2. Создайте корневой документ методом [note.document.add](./note-document-add.md)
3. Получите дерево документов методом [note.document.tree.list](./note-document-tree-list.md), если нужно увидеть структуру страниц внутри базы знаний
4. Если нужна вложенная структура, создавайте дочерние документы тем же методом `note.document.add`, передавая `parentId` родительского документа
5. Читайте отдельный документ методом [note.document.get](./note-document-get.md), если нужно получить его Markdown и служебные поля
6. Уточните состав и типы полей методами `note.document.field.*`, `note.document.tree.field.*` и `note.document.search.field.*`, если строите форму, таблицу или поиск на своей стороне
7. Ищите документы методом [note.document.search.list](./note-document-search-list.md), если известна тема или часть текста
8. Обновляйте заголовок и содержимое методом [note.document.update](./note-document-update.md)
9. Архивируйте документ методом [note.document.archive](./note-document-archive.md), если нужно скрыть его из активной работы, но сохранить структуру
10. Удаляйте документ методом [note.document.delete](./note-document-delete.md), если нужно перенести его и дочерние страницы в корзину

## Ограничения и особенности

- При создании вложенной страницы родительский документ должен принадлежать той же базе знаний. Нельзя создать дочернюю страницу в одной базе знаний и привязать ее к документу из другой
- Размер содержимого `markdown` ограничен `1 048 576` байт. Если документ получается слишком большим, текст нужно сокращать или разбивать на несколько страниц
- Если у документа есть несохраненные изменения совместного редактора, обновление `markdown` может завершиться конфликтом. В таком случае нужно повторить запрос с `overwrite=true`, если содержимое действительно нужно перезаписать
- Архивация документа затрагивает и все дочерние страницы ниже него. Это удобно для архивирования целого раздела, а не одной страницы
- Удаление документа тоже работает каскадно и переносит в корзину все поддерево страниц. Перед удалением стоит учитывать, есть ли у страницы вложенные материалы

## Связь с другими объектами

**Базы знаний.** Документ не существует отдельно от базы знаний. Сначала создают базу знаний, а затем добавляют в нее страницы. Поэтому работа с документами обычно начинается после создания базы знаний в разделе [Базы знаний](../collection/index.md).

**Файлы.** Файлы добавляются не в базу знаний целиком, а в конкретную страницу. Сначала файл загружают методом [note.file.add](../file/note-file-add.md), затем получают готовый блок для вставки методом [note.file.get](../file/note-file-get.md). После этого содержимое страницы обновляют методом [note.document.update](./note-document-update.md), чтобы вложение появилось в тексте документа.
