# Журнал событий: обзор методов

Журнал событий фиксирует действия пользователя в Битрикс24: входы, запросы на смену пароля и ряд системных операций. Доступные события зависят от тарифа Битрикс24. 

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Что такое Журнал событий в Битрикс24](https://helpdesk.bitrix24.ru/open/19070944/)

Методы `main.eventlog.*` позволяют:
- выгрузить события по фильтру, 
- получить конкретную запись,
- настроить регулярное получение новых данных.

## Когда использовать каждый метод

Используйте [main.eventlog.get](./main-eventlog-get.md), когда нужно получить детали конкретного события по ID.

Методы [main.eventlog.list](./main-eventlog-list.md) и [main.eventlog.tail](./main-eventlog-tail.md) похожи, но решают разные задачи.

Используйте [main.eventlog.list](./main-eventlog-list.md), когда нужно:
- сформировать отчет за период,
- найти события по сложным критериям,
- реализовать поиск и навигацию по истории.

Используйте [main.eventlog.tail](./main-eventlog-tail.md), когда нужно:
- настроить мониторинг в реальном времени,
- синхронизировать внешнюю систему с журналом,
- отслеживать новые события после определенной границы.
