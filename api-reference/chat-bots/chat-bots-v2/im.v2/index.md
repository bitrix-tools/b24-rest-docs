# Мессенджер: обзор методов

Методы `im.v2` позволяют получать события мессенджера и работать с файлами в чатах без регистрации отдельного чат-бота.

> Быстрый переход: [все методы](#all-methods)

## Работа с событиями

События мессенджера доставляются в режиме polling: приложение само периодически забирает накопленную очередь.

1. Подпишитесь на запись событий через [im.v2.Event.subscribe](./events/event-subscribe.md).
2. Периодически получайте новые события через [im.v2.Event.get](./events/event-get.md).
3. Передавайте `offset`, чтобы подтверждать уже обработанные события.
4. Для остановки записи используйте [im.v2.Event.unsubscribe](./events/event-unsubscribe.md).

{% note info "Что такое режим polling" %}

Polling — режим получения событий, при котором приложение само периодически запрашивает у сервера накопленные события. Сервер не знает об адресе приложения и ничего не отправляет сам — он только накапливает очередь и отдает ее по запросу.

Это отличает polling от webhook: в webhook-режиме Битрикс24 сам вызывает URL приложения при каждом новом событии. Polling удобен, если у приложения нет постоянного HTTP-сервера или публичного URL.

{% endnote %}

## Работа с файлами

- загрузите файл в чат через [im.v2.File.upload](./files/file-upload.md)
- получите ссылку для скачивания загруженного файла через [im.v2.File.download](./files/file-download.md)
