# События: обзор методов

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

Методы подписывают пользователя на события мессенджера и возвращают их в режиме polling. Используйте их, если интеграции нужно читать поток событий от имени пользователя или приложения.

> Быстрый переход: [все методы](#all-methods)

## Как работать с событиями

1. Включите запись событий через [im.v2.Event.subscribe](./event-subscribe.md).
2. Получайте накопленные события через [im.v2.Event.get](./event-get.md).
3. Передавайте `offset`, чтобы подтверждать уже обработанные записи.
4. Завершите подписку методом [im.v2.Event.unsubscribe](./event-unsubscribe.md).

{% note info "Что такое режим polling" %}

Polling — режим получения событий, при котором приложение само периодически запрашивает у сервера накопленные события. Сервер не знает об адресе приложения и ничего не отправляет сам — он только накапливает очередь и отдает ее по запросу.

Это отличает polling от webhook: в webhook-режиме Битрикс24 сам вызывает URL приложения при каждом новом событии. Polling удобен, если у приложения нет постоянного HTTP-сервера или публичного URL.

{% endnote %}
