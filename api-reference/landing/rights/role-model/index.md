# Ролевая модель прав: обзор методов

Ролевая модель помогает управлять доступом к сайтам в Битрикс24 через роли. Роль объединяет права и тех, кому они назначены: пользователей, группы и отделы. Это помогает не настраивать доступ для каждого пользователя отдельно — права задают один раз и применяют сразу ко всем, кому назначена роль.

Например, можно создать роль «Редактор» и дать право редактировать страницы сайта. Затем назначить роль сотрудникам отдела. Так все сотрудники получат одинаковые права.

## Как назначать роли и настраивать права доступа

Процесс настройки состоит из пяти основных шагов:

1. Проверьте, включена ли ролевая модель прав. Это можно сделать методом [landing.role.isEnabled](../landing-role-is-enabled.md). Если модель выключена, включите ее методом [landing.role.enable](../landing-role-enable.md).

2. Определите тип сайта, для которого нужны роли. При необходимости сверитесь со страницей [Типы сайтов и скоупы](../../types.md).

3. Получите список доступных ролей методом [landing.role.getList](./landing-role-get-list.md) и выберите нужную роль по `id`. После этого назначьте роль пользователям, группам или отделам методом [landing.role.setAccessCodes](./landing-role-set-access-codes.md).

4. Настройте права роли для сайтов методом [landing.role.setRights](./landing-role-set-rights.md). Права можно задать сразу для всех сайтов или только для отдельных. Для этого используйте параметр `rights`: ключ `0` задает права по умолчанию для всех сайтов, а ключ `<siteId>` — права для конкретного сайта.

5. После настройки проверьте результат методом [landing.role.getRights](./landing-role-get-rights.md).

## Связь с другими объектами

Ролевая модель прав работает вместе с сайтами, кодами доступа и режимом управления правами в Битрикс24.

**Сайты.** Права роли настраиваются для определенных сайтов, поэтому для работы нужен идентификатор сайта. Его можно получить методами [landing.site.getList](../../site/landing-site-get-list.md) и [landing.site.add](../../site/landing-site-add.md).

**Коды доступа.** Роль назначают пользователям, группам и отделам через параметр `codes` метода [landing.role.setAccessCodes](./landing-role-set-access-codes.md). Примеры кодов доступа и правила их использования можно посмотреть в описании [landing.site.setRights](../extended-model/landing-site-set-rights.md).

**Модели прав.** В Битрикс24 есть ролевая и [расширенная модель прав](../extended-model/index.md). Одновременно используют только одну из них, поэтому перед настройкой нужно проверить, какая модель сейчас включена. Это можно сделать методом [landing.role.isEnabled](../landing-role-is-enabled.md).
