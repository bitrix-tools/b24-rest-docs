# Права в Сайтах и магазинах: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

В Битрикс24 доступом к сайтам и магазинам можно управлять с помощью расширенной и ролевой моделей прав. С помощью методов можно проверить, какая модель включена, при необходимости переключить ее и настроить доступ нужным способом.

Если права нужно настроить отдельно для конкретного сайта, подойдет [расширенная модель](./extended-model/index.md). В ней доступ задают на уровне сайта. Например, можно разрешить группе сотрудников редактировать один сайт, а для остальных оставить только просмотр. 

Если одинаковые права нужно выдать нескольким пользователям или отделу, подойдет [ролевая модель](./role-model/index.md). В этом случае сначала получают список ролей, назначают роль сотрудникам и после этого настраивают права роли для сайтов. Например, можно создать роль «Редактор», назначить ее отделу и разрешить изменять сайты компании.

Настройки расширенной и ролевой модели хранятся отдельно. При переключении между ними ранее заданные права не удаляются. Когда модель снова включают, ее настройки начинают действовать снова.

{% note warning %}

Права задаются для сайта и действуют на все его страницы. В облачной версии Битрикс24 эта настройка доступна только на платном тарифе. Если перейти на бесплатный тариф, настройка прав отключится автоматически, а все ранее закрытые объекты станут доступны всем пользователям.

{% endnote %}

## Как настроить модель прав

1. Проверьте, какая модель прав сейчас включена, методом [landing.role.isEnabled](./landing-role-is-enabled.md).
2. Если нужно, переключите модель методом [landing.role.enable](./landing-role-enable.md).
3. Получите идентификатор сайта через [landing.site.getList](../site/landing-site-get-list.md).
4. Для расширенной модели настройте права сайта в разделе [Расширенная модель](./extended-model/index.md).
5. Для ролевой модели получите список ролей, назначьте коды доступа и сохраните права в разделе [Ролевая модель](./role-model/index.md).
6. Проверьте результат: для расширенной модели вызовите [landing.site.getRights](./extended-model/landing-site-get-rights.md), для ролевой модели вызовите [landing.role.getRights](./role-model/landing-role-get-rights.md).

## Связь с другими объектами

**Сайты.** Права настраиваются для сайтов, поэтому для работы нужен `siteId`. Его можно получить методами [landing.site.getList](../site/landing-site-get-list.md) и [landing.site.add](../site/landing-site-add.md).

**Страницы.** Права сайта распространяются на все его страницы, поэтому отдельно настраивать доступ для страниц не нужно. 

**Коды доступа и роли.** В ролевой модели доступ назначают не напрямую пользователю, а через роль и коды доступа. Для этого используют методы [landing.role.setAccessCodes](./role-model/landing-role-set-access-codes.md) и [landing.role.setRights](./role-model/landing-role-set-rights.md).
