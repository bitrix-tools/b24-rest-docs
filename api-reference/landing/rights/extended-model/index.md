# Расширенная модель прав: обзор методов

Расширенная модель прав позволяет настраивать доступ отдельно для каждого сайта. В этой модели права задают в настройках сайта — для сотрудников, групп и отделов по их кодам доступа. 

Например, можно дать отделу право редактировать сайт с акциями, а всем остальным сотрудникам — только просматривать его.

> Быстрый переход: [все методы](#all-methods)

## Как настроить права в расширенной модели

Процесс настройки состоит из пяти шагов.

1. Проверьте текущую модель прав методом [landing.role.isEnabled](../landing-role-is-enabled.md). Если включена ролевая модель, переключите на расширенную методом [landing.role.enable](../landing-role-enable.md) со значением `mode: 0`.
2. Получите идентификатор сайта методом [landing.site.getList](../../site/landing-site-get-list.md). 
3. Подготовьте объект `rights` из кодов доступа и кодов операций. 
4. Сохраните новый набор прав методом [landing.site.setRights](./landing-site-set-rights.md). 
5. После настройки проверьте результат под нужным пользователем или в интерфейсе Битрикс24. Метод [landing.site.getRights](./landing-site-get-rights.md) показывает только права текущего пользователя и не возвращает полный список правил, которые сохранены для сайта.

## Связь с другими объектами

Расширенная модель прав связана с сайтами, кодами доступа и выбранной моделью прав в Битрикс24.

**Сайты.** Права настраиваются для определенного сайта, поэтому для работы нужен `siteId`. Его можно получить методами [landing.site.getList](../../site/landing-site-get-list.md) и [landing.site.add](../../site/landing-site-add.md).

**Коды доступа.** Права назначают пользователям, группам и отделам через объект `rights`.

**Модели прав.** В Битрикс24 одновременно работает только одна модель прав. Если доступ нужно настроить для сайта, используйте расширенную модель. Если одинаковые права нужно выдавать через роли, подойдет [ролевая модель прав](../role-model/index.md).
