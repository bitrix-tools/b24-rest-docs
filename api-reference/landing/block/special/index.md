# Специальные блоки: обзор

Специальные блоки в `landing` решают не общую задачу верстки, а отдельные сценарии: навигацию по странице, меню, поиск, карты и встраивание CRM-форм. Поведение блоков зависит от контекста сайта, страницы и доступных объектов.

> Быстрый переход: [все сценарии раздела](#all-scenarios)

## Связанные сценарии

**Манифест блока.** Большая часть специальных возможностей задается в [манифесте](../manifest.md): через секцию `block`, секцию `assets` и настройки атрибутов. Файл манифеста можно получить методом [landing.block.getmanifestfile](../methods/landing-block-get-manifest-file.md).

**Методы блока.** Раздел [Методы работы с блоками](../methods/index.md) описывает инструменты для чтения и изменения содержимого блока. Для изучения готовых реализаций сначала получите список блоков через [landing.block.getrepository](../methods/landing-block-get-repository.md), затем откройте манифест нужного блока через [landing.block.getmanifestfile](../methods/landing-block-get-manifest-file.md).

**Интерактивные блоки.** Если задача связана не с системными подтипами (`subtype`), а с поведением блока в редакторе и на клиентской стороне, посмотрите раздел [Интерактивные блоки](../interactive/index.md).

## Выбор сценария

Откройте подраздел, если нужно:

-  вывести заголовок страницы или навигационную цепочку через системные маркеры — [Навигация и заголовок](./navigation.md),

-  автоматически заполнить блок ссылками меню — [Блоки-меню](./menu.md),

-  добавить поисковую форму, которая отправляет запрос на отдельную страницу — [Формы поиска](./search-forms.md),

-  оформить страницу, на которой выводятся результаты поиска — [Результаты поиска](./search.md),

-  встроить карту и подготовить блок к работе с провайдером карт — [Карты в блоках](./maps.md),

-  встроить CRM-форму в блок — [Формы в блоках](./crm-forms.md).

## Что учитывать при выборе сценария

-  `search-forms` и `search` обычно используются вместе: первая страница описывает форму поиска, вторая — страницу результатов

-  `navigation` обычно дополняет другие сценарии, а `maps`, `crm-forms` и `menu` чаще решают основную задачу блока

-  через `subtype` в манифесте задаются сценарии для [Блоков-меню](./menu.md), [Карт в блоках](./maps.md), [Форм поиска](./search-forms.md) и [Форм в блоках](./crm-forms.md), а через маркеры в HTML — [Навигация и заголовок](./navigation.md).
