# Пользовательские шаблоны: обзор методов

Пользовательские шаблоны позволяют добавить в мастер создания сайтов и страниц собственные шаблоны.

Группа методов `landing.demos.*` помогает работать с пользовательскими шаблонами. В разделе есть методы для регистрации, получения списков и удаления шаблонов. Отдельные методы используют для зарегистрированных и файловых шаблонов.

> Быстрый переход: [все методы](#all-methods)
>
> Пользовательская документация: [Как подготовить пользовательский шаблон](./introduction.md)

## Как начать работу

1. Подготовьте сайт или страницу методами раздела Сайты [разделе Сайты](../site/index.md)
2. Экспортируйте сайт методом [landing.site.fullExport](../site/landing-site-full-export.md)
3. Сохраните экспортированный массив на стороне приложения Битрикс24
4. Передайте экспортированный массив в [landing.demos.register](./landing-demos-register.md)
5. Проверьте результат методом [landing.demos.getList](./landing-demos-get-list.md)

## Ключевые параметры

**XML_ID.** Внешний код зарегистрированного шаблона. Используется при удалении шаблона через [landing.demos.unregister](./landing-demos-unregister.md). Получить его можно методом [landing.demos.getList](./landing-demos-get-list.md). Удаление по коду может затронуть связанные записи шаблона, если они зарегистрированы в приложении с одним и тем же кодом.

**type.** Тип шаблона. Его указывают при регистрации, а также при вызове методов [landing.demos.getSiteList](./landing-demos-get-site-list.md) и [landing.demos.getPageList](./landing-demos-get-page-list.md). Например: `page`, `store`, `knowledge`.

**tpl_type.** Тип использования шаблона в мастере. Параметр используют при регистрации шаблона. Возможны два значения: `S` — шаблон сайта, `P` — шаблон страницы.

## Связь с другими объектами

**Сайт.** Пользовательский шаблон создают на основе готового сайта или страницы методами [раздела Сайты](../site/index.md). Для этого сайт сначала экспортируют методом [landing.site.fullExport](../site/landing-site-full-export.md). Затем экспортированный массив передают в [landing.demos.register](./landing-demos-register.md).

**Локализация.** Название, описание и строковые значения в поддерживаемых полях шаблона можно локализовать при регистрации. Для этого используют параметры `lang` и `lang_original`. Подробнее в статье [Локализация шаблона](./localization.md).
