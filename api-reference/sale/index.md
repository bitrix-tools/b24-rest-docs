# Интернет-магазин: обзор разделов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Интернет-магазин в Битрикс24 — это готовое решение для онлайн-продаж. Он нужен, чтобы быстро запустить прием заказов, управлять каталогом, оплатой и доставкой в единой системе.

В разделах Интернет-магазина собраны основные сценарии:

- создание и изменение заказов
- управление корзиной и составом отгрузок
- настройка свойств заказа и условий их показа
- учет оплат, статусов и кассовых операций
- интеграция со службами доставки и обработка событий

## Как начать работу

1. Проверьте структуру полей и типов на странице [Типы данных и структура объектов в REST API Интернет-магазина](./data-types.md).
2. Проверьте права доступа и роль пользователя перед выполнением изменений.
3. Определите основной объект сценария: заказ, корзина, оплата, отгрузка или свойство.
4. Получите рабочие идентификаторы через `list` или `get` в профильном разделе.
5. Выполните изменение объекта методами `add`, `update` или `delete`.
6. При необходимости подпишитесь на [События](./events/index.md), чтобы отслеживать изменения в реальном времени.

## Права доступа

Настройка прав доступа доступна не на всех тарифах. По умолчанию настроить права может администратор Битрикс24, но он может выдать эти права другим сотрудникам.

Права доступа для разделов Сайты и Магазины общие. Если изменить их в одном разделе, изменения применятся и к другому.

## Связь с другими объектами

**CRM.** Заказы и оплаты Интернет-магазина используются в CRM-сценариях, где важны состав заказа, стоимость и статусы. Для работы с заказами используйте раздел [Заказ](./order/index.md), для оплат — [Оплаты](./payment/index.md), для отслеживания изменений — [События](./events/index.md).

**Торговый каталог.** Интернет-магазин использует товарные данные из каталога: товары, цены, свойства и остатки. Для этих сценариев используйте разделы [Торговый каталог](../catalog/index.md), [Товары](../catalog/product/index.md) и [Цена](../catalog/price/index.md).

**Службы доставки.** Доставка связана с отгрузками, свойствами отгрузки и транспортными заявками в разделе [Службы доставки](./delivery/index.md).
