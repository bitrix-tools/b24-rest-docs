# Взаимодействие с UI: обзор методов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Раздел описывает клиентские методы и сценарии, через которые приложение взаимодействует с интерфейсом Битрикс24 без REST-запроса к серверу. Методы работают в контексте текущего плейсмента и обмениваются данными через `postMessage`.

Через этот раздел можно получить контекст текущего плейсмента, узнать список доступных команд и событий, подписаться на события интерфейса и вызвать команды UI. Для отдельных плейсментов доступны специализированные интерфейсы с дополнительными командами и событиями.

> Быстрый переход: [все методы](#all-methods)

## Как начать работу

1. Откройте приложение в нужном плейсменте
2. Получите контекст вызова через [BX24.placement.info](./bx24-placement-info.md)
3. Запросите доступные команды и события через [BX24.placement.getInterface](./bx24-placement-get-interface.md)
4. Подписывайтесь на события через [BX24.placement.bindEvent](./bx24-placement-bind-event.md) и вызывайте команды через [BX24.placement.call](./bx24-placement-call.md)
5. Если нужно открыть стандартную страницу Битрикс24, отдельный слайдер приложения или системный диалог, используйте [дополнительные методы BX24.js](../bx24-widget-methods.md) и [системные диалоги](../../../sdk/bx24-js-sdk/system-dialogues/index.md)

## Ограничения и особенности

- Методы этого раздела работают на стороне интерфейса и не заменяют REST API
- Перед вызовом специальных команд лучше сначала проверить доступный интерфейс через [BX24.placement.getInterface](./bx24-placement-get-interface.md)
- Команда интерфейса работает только там, где она поддерживается. Один и тот же вызов может быть валиден для `CALL_CARD` и недоступен в другом плейсменте
- Методы [BX24.openPath](../bx24-widget-methods.md), [BX24.openApplication](../bx24-widget-methods.md) и [BX24.closeApplication](../bx24-widget-methods.md) управляют навигацией и окнами интерфейса, но не заменяют получение или сохранение данных через REST API

## Специальные возможности отдельных плейсментов

Некоторые плейсменты поддерживают не только базовые методы `BX24.placement.*`, но и специальный JS-интерфейс для управления конкретными элементами Битрикс24.

|| **Плейсмент** | **Что доступно** ||
|| [Карточка звонка `CALL_CARD`](./call-card/index.md) | Методы получения состояния звонка, управления автозакрытием карточки и события карточки звонка ||
|| [Карточка CRM](#crm-card) | Дополнительная команда интерфейса `reloadData` для обновления данных формы ||
|| [Фоновый плейсмент `PAGE_BACKGROUND_WORKER`](./page-background-worker/index.md) | События карточки звонка и методы управления ее интерфейсом из фоновой встройки ||
|#

## Карточка CRM {#crm-card}

В контексте карточки CRM доступна дополнительная команда интерфейса `reloadData`. Ее вызывают через [BX24.placement.call](./bx24-placement-call.md), чтобы обновить данные формы в интерфейсе после действий приложения.

```js
BX24.placement.call('reloadData', {}, function () {
    console.log('reloadData called');
});
```

Команда обновляет данные только в интерфейсе карточки. Она не сохраняет изменения в CRM автоматически. Чтобы данные попали в базу, пользователь должен сохранить карточку.

Перед вызовом команды рекомендуется проверить доступный интерфейс через [BX24.placement.getInterface](./bx24-placement-get-interface.md).
