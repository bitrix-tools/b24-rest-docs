# Карточка звонка CALL_CARD: обзор методов и событий

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Плейсмент `CALL_CARD` используют приложения, которые встраиваются в карточку звонка в CRM. Через js-интерфейс плейсмента приложение может получить данные текущего звонка, управлять автозакрытием карточки и реагировать на изменения клиента или состояния звонка.

> Быстрый переход: [все методы и события](#all-methods)
>
> Пользовательская документация: [Карточка звонка](https://helpdesk.bitrix24.ru/open/18425138/)

## Как начать работу с CALL_CARD

1. Откройте приложение в плейсменте `CALL_CARD`
2. Получите список доступных команд и событий через [BX24.placement.getInterface](../bx24-placement-get-interface.md)
3. Вызывайте команды плейсмента через [BX24.placement.call](../bx24-placement-call.md)
4. Подписывайтесь на события карточки звонка через [BX24.placement.bindEvent](../bx24-placement-bind-event.md)

## Как устроен вызов API плейсмента

В `CALL_CARD` команды и события плейсмента не являются самостоятельными REST-методами. Приложение работает с ними через общий js-интерфейс встраивания.

Чтобы выполнить команду плейсмента, передайте ее имя в [BX24.placement.call](../bx24-placement-call.md). Для `CALL_CARD` это команды `getStatus`, `disableAutoClose` и `enableAutoClose`. Команда `getStatus` возвращает данные текущего звонка, а команды управления автозакрытием возвращают пустой массив при успешном вызове.

Чтобы реагировать на изменения в карточке без повторного запроса, подпишитесь на события через [BX24.placement.bindEvent](../bx24-placement-bind-event.md). В `CALL_CARD` доступны события `CallCard::EntityChanged`, `CallCard::CallStateChanged` и `CallCard::BeforeClose`. В обработчик событий могут приходить данные клиента, состояние звонка с дополнительными параметрами или вызов без данных, если карточка закрывается.
