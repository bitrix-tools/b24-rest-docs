# Автозаполнение реквизитов в карточке CRM

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

> Scope: [`crm`](../../../scopes/permissions.md)

Точки автозаполнения реквизитов подключают приложение к поиску данных в карточке CRM. Приложение работает как внешний источник: получает строку поиска, возвращает найденные варианты и передает данные выбранного варианта для подстановки в карточку.

Код точки встраивания передается в параметре `PLACEMENT` метода [placement.bind](../../placement-bind.md). Если приложение работает только с отдельными странами, передайте их идентификаторы через запятую без пробелов в параметре `OPTIONS[countries]`.

Битрикс24 вызывает внешний поиск, когда пользователь ввел в строку поиска не менее трех символов.

Команды `crmShowFoundEntities` и `crmShowCreatedEntity` вызываются через `BX24.placement.call`, а событие `onCrmEntityIsNeedToCreate` обрабатывается через `BX24.placement.bindEvent`. Отдельно регистрировать их не нужно.

{% note info "" %}

Обработчик не будет доступен в интерфейсе выбора источника поиска, пока установка приложения не завершена. [Проверьте установку приложения](../../../../settings/app-installation/installation-finish.md)

{% endnote %}

## Как выбрать точку встраивания

#|
|| **Точка встраивания** | **Код** | **Когда использовать** ||
|| [Автозаполнение реквизитов клиента](./requisite-autocomplete.md) | `CRM_REQUISITE_AUTOCOMPLETE` | Для поиска и подстановки реквизитов компании или контакта ||
|| [Автозаполнение банковских реквизитов](./bank-detail-autocomplete.md) | `CRM_BANK_DETAIL_AUTOCOMPLETE` | Для поиска и подстановки банковских реквизитов, например по БИК ||
|#

## Порядок работы

1. Пользователь вводит строку в поле автозаполнения реквизитов
2. Битрикс24 вызывает обработчик приложения и передает строку поиска в `PLACEMENT_OPTIONS.searchQuery`
3. Приложение передает найденные варианты командой `crmShowFoundEntities`
4. Если пользователь выбирает вариант, приложение получает событие `onCrmEntityIsNeedToCreate` и передает данные для подстановки командой `crmShowCreatedEntity`

## Типовые ошибки

#|
|| **Ошибка** | **Как исправить** ||
|| Обработчик зарегистрирован, но недоступен при поиске | Проверьте, что приложение установлено, а в `PLACEMENT` передан код нужной точки встраивания ||
|| Обработчик недоступен для нужной страны | Проверьте значение `OPTIONS[countries]`. Строка должна содержать идентификаторы стран через запятую без пробелов ||
|| Найденные варианты не отображаются | Передавайте массив вариантов в поле `data` команды `crmShowFoundEntities` ||
|| После выбора вариант не подставляется в карточку | Подпишитесь на `onCrmEntityIsNeedToCreate` и после создания вызовите `crmShowCreatedEntity` ||
|#
