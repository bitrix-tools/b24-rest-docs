# Универсальные виджеты: обзор точек встраивания

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Раздел описывает универсальные точки встраивания, которые не привязаны к одному конкретному интерфейсу Битрикс24. Через них приложение может открываться по специальной ссылке в контенте или работать в фоне на всех страницах портала.

Для регистрации виджета используйте метод [placement.bind](../placement-bind.md) и передавайте нужный код в параметре `PLACEMENT`.

> Быстрый переход: [все виджеты](#all-widgets)

## Как выбрать точку встраивания

- Пользователь должен открыть приложение по специальной ссылке в сообщении, комментарии, задаче или другом тексте с контентом — [REST_APP_URI](./app-url.md)
- Приложение должно работать в фоне на всех страницах портала и реагировать на события без отдельного видимого элемента интерфейса — [PAGE_BACKGROUND_WORKER](./background-worker.md)

## Как начать работу

1. Определите сценарий запуска: открытие по ссылке или фоновая работа на страницах портала
2. Зарегистрируйте обработчик через [placement.bind](../placement-bind.md) и передайте подходящий `PLACEMENT`
3. Разберите `PLACEMENT_OPTIONS` в обработчике, чтобы получить контекст запуска
4. Если виджет должен открывать интерфейс приложения, используйте [методы JavaScript для виджетов](../bx24-widget-methods.md)
5. Если сценарий требует обмена сигналами между backend и frontend, подключите [интерактивное взаимодействие](../../../settings/interactivity/index.md)

## Особенности универсальных виджетов

**REST_APP_URI.** Виджет открывается по ссылке формата `/marketplace/view/#APP_CODE#/` и может принимать произвольные параметры через `params`. Такой сценарий подходит, когда приложение нужно запускать из текста или другого контента, где можно разместить ссылку.

**PAGE_BACKGROUND_WORKER.** Виджет загружается на всех страницах портала без отдельного визуального элемента. Обработчик должен отвечать быстро, иначе Битрикс24 может отключить его и уведомить приложение через `OPTIONS[errorHandlerUrl]`.

## Что получает обработчик

Данные передаются в виде POST-запроса {.b24-info}

{% list tabs %}

- REST_APP_URI

    ```php
    Array
    (
        [DOMAIN] => xxx.bitrix24.com
        [PROTOCOL] => 1
        [LANG] => en
        [APP_SID] => 195ec4ee87932d8f9bbbd6a2f0a83553
        [AUTH_ID] => f27bbb6600705a0700005a4b00000001f0f107398c3f17f5fc48d5ce194d5c65de7cfb
        [AUTH_EXPIRES] => 3600
        [REFRESH_ID] => e2fae26600705a0700005a4b00000001f0f1075f986dbd8dff24c36c2ad9bb0816a665
        [member_id] => da45a03b265edd8787f8a258d793cc5d
        [status] => L
        [PLACEMENT] => REST_APP_URI
        [PLACEMENT_OPTIONS] => {"test":"y"}
    )
    ```

- PAGE_BACKGROUND_WORKER

    ```php
    Array
    (
        [handler] => 1
        [DOMAIN] => restapi.bitrix24.ru
        [PROTOCOL] => 1
        [LANG] => ru
        [APP_SID] => 588b8a98e848778a4ffb38fbcf70f2b9
        [AUTH_ID] => 4172bb6600705a0700005a4b00000001f0f107c42ca5bd5f61030c5d9c3e4d60d11b5a
        [AUTH_EXPIRES] => 3600
        [REFRESH_ID] => 31f1e26600705a0700005a4b00000001f0f107b1918506d8a2ed9ecf76e8fdac962471
        [member_id] => da45a03b265edd8787f8a258d793cc5d
        [status] => L
        [PLACEMENT] => PAGE_BACKGROUND_WORKER
        [PLACEMENT_OPTIONS] => {"ID":"PAGE_BACKGROUND_WORKER","URI":"\/company\/personal\/user\/1\/blog\/"}
    )
    ```

{% endlist %}

### PLACEMENT_OPTIONS

Значение `PLACEMENT_OPTIONS` передается как JSON-строка с контекстом запуска.

#|
|| **Виджет** | **Ключи** | **Описание** ||
|| [REST_APP_URI](./app-url.md) | Произвольные ключи из `params` | Параметры, которые приложение передает в специальной ссылке формата `/marketplace/view/#APP_CODE#/` ||
|| [PAGE_BACKGROUND_WORKER](./background-worker.md) | `ID`, `URI` | Код виджета и адрес текущей страницы, на которой был запущен обработчик ||
|#

## Связь с другими объектами

**Слайдер приложения.** Для сценариев, где после фонового сигнала нужно открыть интерфейс приложения, используйте [методы JavaScript для виджетов](../bx24-widget-methods.md).

**Интерактивность.** Виджет `PAGE_BACKGROUND_WORKER` часто используют вместе с механизмом [интерактивного взаимодействия](../../../settings/interactivity/index.md), чтобы передавать сигналы между backend и frontend без ручного запуска со стороны пользователя.
