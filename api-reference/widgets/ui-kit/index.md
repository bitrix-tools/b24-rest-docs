# Битрикс24 UI Kit: обзор

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../../../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../../../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

Битрикс24 UI Kit — библиотека компонентов, дизайн-токенов и иконок для интерфейсов приложений Битрикс24. Она помогает собрать экран приложения так, чтобы он выглядел и вел себя как часть Битрикс24.

Библиотеку используют для страниц приложений, виджетов и встроенных интерфейсов: форм, списков, кнопок, модальных окон и других элементов. Компоненты не обращаются к Битрикс24 и не управляют правами пользователя. Информацию для экрана и проверку доступа приложение получает отдельно: через методы Битрикс24, серверную часть или SDK.

> Полная документация библиотеки: [Битрикс24 UI-Kit](https://bitrix24.github.io/b24ui/)

## Как начать работу

1. Определите, где пользователь откроет экран: на странице приложения, в виджете или в слайдере. Для экрана внутри Битрикс24 выберите [место встраивания](../placements.md)
2. Подключите UI Kit по инструкции [Быстрый старт](./quick-start.md) или по документации для [Nuxt](https://bitrix24.github.io/b24ui/docs/getting-started/installation/nuxt/) и [Vue/Vite](https://bitrix24.github.io/b24ui/docs/getting-started/installation/vue/)
3. Соберите экран из Vue-компонентов, дизайн-токенов Tailwind CSS или иконок. Готовые элементы смотрите в статье [Компоненты и шаблоны](./components.md)
4. Настройте получение данных через методы Битрикс24, серверную часть или SDK. Подробнее — в статье [Интеграция UI Kit с REST API и бизнес-логикой](./app-logic.md)
5. Для экрана в виджете зарегистрируйте обработчик методом [placement.bind](../placement-bind.md): передайте код места встраивания в параметре `PLACEMENT` и внешний URL страницы приложения в параметре `HANDLER`
6. Если обработчик регистрируется во время установки приложения, завершите установку по инструкции [Завершение установки приложений](../../../settings/app-installation/installation-finish.md)

## Связь с другими объектами

**Приложение.** UI Kit используется внутри интерфейса приложения. Способ установки и доступность интерфейса зависят от настроек [приложения](../../../settings/app-installation/index.md).

**Виджеты.** Если экран открыт из виджета, UI Kit оформляет страницу обработчика. Регистрацией, просмотром и удалением обработчиков управляют методы [placement.bind](../placement-bind.md), [placement.get](../placement-get.md) и [placement.unbind](../placement-unbind.md).

**Данные Битрикс24.** Когда экрану нужны данные, приложение обращается к методам того инструмента Битрикс24, с которым работает интерфейс.

**BX24 SDK.** Внутри виджета можно открывать интерфейс приложения, переходить к стандартным страницам Битрикс24 и закрывать окно приложения методами [BX24 SDK для виджетов](../bx24-widget-methods.md).

## Что важно учитывать

- UI Kit не требует отдельного scope: права проверяются для конкретного действия приложения
- Для регистрации, просмотра и удаления обработчика scope проверяйте в методах [placement.bind](../placement-bind.md), [placement.get](../placement-get.md) и [placement.unbind](../placement-unbind.md)
- Для получения списка доступных мест встраивания scope проверяйте в методе [placement.list](../placement-list.md)
- Для методов работы с данными scope проверяйте на странице конкретного метода. Общий список значений смотрите в справочнике [Права доступа](../../scopes/permissions.md)
- Обработчик виджета должен быть доступен по внешнему URL, если он регистрируется методом [placement.bind](../placement-bind.md)
- Пока установка приложения не завершена, зарегистрированные виджеты видят администраторы или пользователи с правом на установку приложений
