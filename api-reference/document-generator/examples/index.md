# Частые кейсы и сценарии генератора документов: обзор кейсов

{% note tip "" %}

Если вы разрабатываете интеграции для Битрикс24 с помощью AI-инструментов (Codex, Claude Code, Cursor), подключите [MCP-сервер](../../../sdk/mcp.md), чтобы ассистент использовал официальную REST-документацию.

{% endnote %}

Шаблон в генераторе документов — это файл `.docx` с плейсхолдерами полей, которые при создании документа заменяются данными из запроса. Плейсхолдеры оформляются в фигурных скобках, например `{DocumentNumber}` или `{MyField}`. Подробнее в статье [{#T}](../templates/index.md).

Кейсы помогают подготовить данные для создания документа по шаблону методом [documentgenerator.document.add](../document-generator-document-add.md). После создания документа получить ссылки на файлы и продолжить работу с результатом можно методами раздела [{#T}](../index.md).

> Быстрый переход: [все кейсы](#all-cases)

## С чего начать

1. Подготовьте шаблон `.docx` с плейсхолдерами полей
2. Получите карту полей шаблона методом [documentgenerator.template.getfields](../templates/document-generator-template-get-fields.md)
3. Определите, какие данные нужно передать в `values` и `fields` метода [documentgenerator.document.add](../document-generator-document-add.md)
   - `values` — что подставить в документ
   - `fields` — как интерпретировать эти данные: тип поля, формат, изображение, печать, таблица или повторяющийся блок
4. Откройте пример, который ближе всего к вашему сценарию, и адаптируйте его под свой шаблон
