# Регионы в генераторе документов: обзор методов

Методы раздела позволяют получать список регионов, просматривать регион по коду или идентификатору, а также создавать, изменять и удалять пользовательские регионы.

> Быстрый переход: [все методы](#all-methods)

## Что определяет регион

Каждый шаблон генератора документов привязывается к региону. Регион влияет на то, как формируются данные в документе:

- язык региона `languageId`
- формат даты `formatDate`
- формат даты и времени `formatDatetime`
- формат полного имени `formatName`
- набор служебных фраз `phrases`, которые используются при генерации текста

Предустановленные регионы хранятся в системе по коду региона, например `ru`. Пользовательские регионы создаются через REST и получают числовой идентификатор. Метод [documentgenerator.region.get](./document-generator-region-get.md) принимает и код региона, и `id`.

## Связь с шаблонами

Регион используется в шаблонах генератора документов и определяет локальные правила формирования текста и значений полей.

Для работы с шаблонами используйте методы раздела [Шаблоны](../templates/index.md). После изменения параметров региона новые настройки применяются при последующей генерации документов в шаблонах этого региона.

## Особенности работы

Нельзя удалить регион, если к нему привязаны активные шаблоны.

## Предустановленные регионы

В генераторе документов доступны следующие предустановленные регионы:

- `ru` — Россия
- `by` — Беларусь
- `kz` — Казахстан
- `ua` — Украина
- `br` — Бразилия
- `sp` — Испания
- `mx` — Мексика
- `de` — Германия
- `fr` — Франция
- `uk` — Великобритания
- `pl` — Польша
