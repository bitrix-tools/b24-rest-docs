# Локальные интеграции: сценарии использования

{% note tip "" %}

Выберите инструмент для разработки с AI-агентом:

- используйте [Битрикс24 Вайбкод](../ai-tools/vibecode.md), чтобы создать приложение для Битрикс24 по описанию задачи без знания языков программирования. Агент напишет код и разместит приложение на сервере без ручной настройки хостинга
- используйте [MCP-сервер](../ai-tools/mcp.md), чтобы разрабатывать интеграцию через REST API в своем проекте. Агент будет обращаться к официальной REST-документации

{% endnote %}

Сценарий описывает практическую задачу и порядок методов для ее выполнения. В локальных интеграциях сценарии показывают, как передать данные в CRM, создать задачу с файлом, встроить интерфейс приложения и настроить чат-бота.

> Быстрый переход: [все сценарии](#choose-tutorial)
>
> Пользовательская документация: [как создать вебхуки и приложения для Битрикс24](https://helpdesk.bitrix24.ru/open/20886106/)

## Связь с другими объектами

Сценарии связаны с объектами CRM, задачами, файлами Диска, интерфейсом Битрикс24 и чатами.

- **CRM.** Данные из внешней формы можно сохранить в контакте методом [crm.contact.add](../api-reference/crm/contacts/crm-contact-add.md). Для переноса существующих данных используйте методы [crm.item.import](../api-reference/crm/universal/import/crm-item-import.md) и [crm.item.batchImport](../api-reference/crm/universal/import/crm-item-batch-import.md)
- **Задачи и файлы Диска.** Файл сначала загружают методом [disk.folder.uploadfile](../api-reference/disk/folder/disk-folder-upload-file.md), затем его идентификатор передают при создании задачи методом [tasks.task.add](../api-reference/tasks/tasks-task-add.md)
- **Интерфейс Битрикс24.** Виджет открывает страницу приложения внутри карточки CRM. Для регистрации обработчика используют метод [placement.bind](../api-reference/widgets/placement-bind.md). Метод работает только в контексте приложения
- **Чаты и чат-боты.** Чат-бота регистрируют методом [imbot.v2.Bot.register](../api-reference/chat-bots/chat-bots-v2/imbot.v2/bots/bot-register.md). Для получения событий можно настроить периодический опрос или указать URL обработчика. Методы [imbot.v2](../api-reference/chat-bots/chat-bots-v2/index.md#all-methods) доступны через входящий вебхук и локальное приложение

Подробнее об инструментах читайте в статьях [Входящие и исходящие вебхуки](./local-webhooks.md) и [Локальные приложения](./local-apps.md).

## Как начать работу

1. Определите задачу интеграции
2. Выберите сценарий и инструмент в [таблице](#choose-tutorial)
3. Проверьте `scope` методов и права пользователя, от имени которого интеграция выполняет запросы
4. Подготовьте данные и идентификаторы. Если Битрикс24 должен открыть страницу приложения или отправить ей событие, подготовьте обработчик — страницу с URL, доступным из внешней сети
5. Выполните методы в указанном порядке и проверьте результат
